## Result Ordering

### orderBy()

Sorts by a column, ascending by default:

```typescript
// Ascending (default)
const users = await db.table('users')
  .orderBy('name')
  .get();

// Descending
const latest = await db.table('posts')
  .orderBy('created_at', 'DESC')
  .get();
```

The direction must be `'ASC'` or `'DESC'` (case-insensitive) — anything else throws:

```
Error: Invalid direction: down. Use 'ASC' or 'DESC'.
```

### Multi-Column Ordering

Chain `orderBy()` calls; they apply in order:

```typescript
const products = await db.table('products')
  .orderBy('category_id', 'ASC')
  .orderBy('price', 'DESC')
  .get();
// ORDER BY category_id ASC, price DESC
```

### Ordering Joined Columns

```typescript
const orders = await db.table('orders')
  .select(['orders.id', 'users.name', 'orders.total'])
  .join('users', 'orders.user_id', '=', 'users.id')
  .orderBy('users.name', 'ASC')
  .orderBy('orders.total', 'DESC')
  .get();
```

## Grouping

### groupBy()

```typescript
const byCountry = await db.table('users')
  .select(['country'])
  .groupBy('country')
  .orderBy('country', 'ASC')
  .get();
// One row per country
```

Chain multiple calls for composite grouping:

```typescript
const rows = await db.table('orders')
  .select(['status', 'currency'])
  .groupBy('status')
  .groupBy('currency')
  .get();
```

## Limiting Results

### limit()

```typescript
const top5 = await db.table('products')
  .orderBy('sales', 'DESC')
  .limit(5)
  .get();
```

### offset()

For explicit row offsets without page math:

```typescript
const rows = await db.table('logs')
  .orderBy('id', 'ASC')
  .limit(50)
  .offset(100)    // skip the first 100 rows
  .get();
```

## Pagination

### paginate() — The One-Call Option

For UIs, prefer `paginate()` — items + metadata in a single call (see [Data Retrieval](/guides/query-builder/data-retrieval)):

```typescript
const { items, total, totalPages, hasNext, hasPrev } = await db.table('products')
  .where('published', '=', true)
  .orderBy('id', 'ASC')
  .paginate(2, 25);
```

### page()

`page(n)` works together with `limit()`: the offset is computed as `(n - 1) * limit`.

```typescript
const pageSize = 20;

// Page 1: rows 1–20
const p1 = await db.table('products').orderBy('id', 'ASC').limit(pageSize).page(1).get();

// Page 3: rows 41–60
const p3 = await db.table('products').orderBy('id', 'ASC').limit(pageSize).page(3).get();
```

::callout{type="warning"}
**Call `limit()` before `page()`.** Without a limit in the chain, `page()` cannot compute an offset and is ignored.
::

### Complete Pagination Helper

```typescript
interface Paginated<T> {
  items: T[];
  page: number;
  perPage: number;
  total: number;
  totalPages: number;
  hasNext: boolean;
  hasPrev: boolean;
}

async function paginateProducts(page: number, perPage = 20): Promise<Paginated<any>> {
  const items = await db.table('products')
    .where('published', '=', true)
    .orderBy('id', 'ASC')
    .limit(perPage)
    .page(page)
    .get();

  // count() ignores limit/page — it counts the full filtered set
  const total = await db.table('products')
    .where('published', '=', true)
    .count();

  const totalPages = Math.ceil(total / perPage);

  return {
    items,
    page,
    perPage,
    total,
    totalPages,
    hasNext: page < totalPages,
    hasPrev: page > 1,
  };
}
```

### Stable Pagination Rules

1. **Always order paginated queries** — without `orderBy()` the database may return rows in different order per request, making pages overlap or skip.
2. **Order by a unique column (or end with one)** — `orderBy('created_at', 'DESC')` plus `orderBy('id', 'DESC')` breaks ties deterministically.
3. **Use the same filters for items and count** — otherwise `totalPages` lies.

```typescript
const rows = await db.table('events')
  .orderBy('created_at', 'DESC')
  .orderBy('id', 'DESC')        // tie-breaker
  .limit(50)
  .page(page)
  .get();
```

## Practical Examples

### Leaderboard

```typescript
const top10 = await db.table('players')
  .select(['id', 'username', 'score'])
  .orderBy('score', 'DESC')
  .orderBy('id', 'ASC')
  .limit(10)
  .get();
```

### Infinite Scroll Feed

```typescript
async function feedPage(page: number) {
  return db.table('posts')
    .select(['id', 'title', 'created_at'])
    .where('status', '=', 'published')
    .orderBy('created_at', 'DESC')
    .orderBy('id', 'DESC')
    .limit(15)
    .page(page)
    .get();
}
```

### Admin Table with Sortable Columns

```typescript
const allowedSorts = ['name', 'email', 'created_at'];

async function listUsers(sortBy: string, dir: 'ASC' | 'DESC', page: number) {
  const column = allowedSorts.includes(sortBy) ? sortBy : 'created_at';

  return db.table('users')
    .select(['id', 'name', 'email', 'created_at'])
    .orderBy(column, dir)
    .limit(25)
    .page(page)
    .get();
}
```

Whitelisting sortable columns keeps user input from reaching the query untrusted.
