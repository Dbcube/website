While the [Dbcube language extension](/vscode-extension/overview) covers `.cube` files, **Dbcube Snippets** (published as **Dbcube Snippets** — id `Dbcube.dbcube-snippets`) supercharges your TypeScript and JavaScript when writing queries: contextual autocompletion, live parameter hints, rich hover documentation, and a library of ready-made patterns for the whole query builder (including `whereGroup`, `batch`, `useComputes` and `useTriggers`).

## Installation

### From the Marketplace

1. Open VS Code → Extensions (`Ctrl+Shift+X`)
2. Search **"Dbcube Snippets"**
3. Click **Install**

Or from the command line (also on Open VSX for Cursor/Windsurf):

```bash
code --install-extension Dbcube.dbcube-snippets
```

### From VSIX

```bash
code --install-extension dbcube-snippets-1.0.0.vsix
```

Supported languages: **TypeScript** (`.ts`, `.tsx`) and **JavaScript** (`.js`, `.jsx`).

## Features

### Intelligent Autocompletion

The extension understands the fluent interface and suggests the right thing at each step of the chain:

```typescript
const users = await db.table('users')
  .select(['id', 'name'])      // ← suggests column arrays
  .where('status', '=', '')    // ← suggests operators (=, !=, >, LIKE, IS NULL…)
  .orderBy('created_at', '')   // ← suggests ASC / DESC
  .get();                      // ← suggests execution methods
```

### Smart NULL Handling

It knows the TypeScript overloads — `IS NULL` / `IS NOT NULL` take no third argument:

```typescript
.where('deleted_at', 'IS NULL')      // ✓ no value parameter suggested
.where('status', '=', 'active')      // ✓ value parameter expected
```

### Hover Documentation & Parameter Hints

Hover any query builder method for a full description, parameter types, and usage examples. Signature help follows you while typing arguments.

## Snippet Library

Type a prefix and press `Tab`:

| Prefix | Generates |
|--------|-----------|
| `dbcube-db` | ORM init + database connection |
| `dbcube-select` | SELECT with column list |
| `dbcube-select-where` | SELECT with WHERE |
| `dbcube-first` | Fetch a single record |
| `dbcube-insert` | INSERT records |
| `dbcube-update` | UPDATE with condition |
| `dbcube-delete` | DELETE with condition |
| `dbcube-where-null` / `dbcube-where-not-null` | NULL conditions |
| `dbcube-or-where` | OR condition |
| `dbcube-join` / `dbcube-left-join` | JOIN clauses |
| `dbcube-order-by` / `dbcube-limit` | Ordering and limiting |
| `dbcube-count` | COUNT aggregation |
| `dbcube-auth-user` | Complete login query pattern |
| `dbcube-paginate` | Pagination (page + total count) |
| `dbcube-search` | Multi-column LIKE search |

Example — `dbcube-paginate` expands to a full pagination block:

```typescript
const page = 1;
const perPage = 20;

const items = await db.table('products')
  .where('status', '=', 'active')
  .orderBy('created_at', 'DESC')
  .limit(perPage)
  .page(page)
  .get();

const total = await db.table('products')
  .where('status', '=', 'active')
  .count();
```

## Configuration

```json
{
  "dbcubeSnippets.enableAutoCompletion": true,
  "dbcubeSnippets.enableHoverDocumentation": true,
  "dbcubeSnippets.enableParameterHints": true,
  "dbcubeSnippets.suggestionLevel": "intermediate"
}
```

| Setting | Default | Description |
|---------|---------|-------------|
| `enableAutoCompletion` | `true` | Contextual method suggestions |
| `enableHoverDocumentation` | `true` | Docs on hover |
| `enableParameterHints` | `true` | Live signature help |
| `suggestionLevel` | `"intermediate"` | `basic`, `intermediate`, or `advanced` |

## Commands

Available from the Command Palette (`Ctrl+Shift+P`):

- **Dbcube: Insert Query Template** — drops a base query scaffold at the cursor
- **Dbcube: Show Documentation** — opens this documentation site

## Troubleshooting

- **Hints not appearing?** Restart the TypeScript server: `Ctrl+Shift+P` → "TypeScript: Restart TS Server".
- **Suggestions too aggressive/quiet?** Tune `dbcubeSnippets.suggestionLevel`.
- Autocompletion works best with explicit Dbcube imports in the file.
