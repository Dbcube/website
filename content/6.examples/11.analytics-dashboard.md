The read-heavy side of an app: the numbers behind a dashboard. Aggregations,
grouped breakdowns and exports that don't blow up memory — all typed.

## KPI tiles (the big numbers)

Aggregations execute immediately and return a `number` — no `.get()` needed.

```ts
import { dbcube } from "dbcube";

const db = dbcube.database("shop");

export async function kpis() {
  const [orders, revenueCents, avgOrder, customers] = await Promise.all([
    db.table("orders").where("status", "=", "paid").count(),
    db.table("orders").where("status", "=", "paid").sum("total_cents"),
    db.table("orders").where("status", "=", "paid").avg("total_cents"),
    db.table("customers").count(),
  ]);

  return {
    orders,
    revenue: revenueCents / 100,
    avgOrderValue: avgOrder / 100,
    customers,
  };
}
```

## Breakdown by category (groupBy + having)

```ts
interface CategoryRow { category: string; orders: number; revenue_cents: number; }

export async function revenueByCategory() {
  return db.table<CategoryRow>("order_items")
    .join("products", "order_items.product_id", "=", "products.id")
    .selectRaw([
      "products.category AS category",
      "COUNT(*) AS orders",
      "SUM(order_items.unit_price_cents * order_items.qty) AS revenue_cents",
    ])
    .groupBy("products.category")
    .having("orders", ">", 0)
    .orderBy("revenue_cents", "DESC")
    .get();
}
```

## A time series (orders per day, last 30 days)

Use `raw()` for engine-specific date functions — typed via `raw<T>()`.

```ts
interface DailyPoint { day: string; orders: number; revenue_cents: number; }

export async function ordersPerDay() {
  return db.raw<DailyPoint>(`
    SELECT DATE(created_at)               AS day,
           COUNT(*)                       AS orders,
           SUM(total_cents)               AS revenue_cents
    FROM orders
    WHERE status = ? AND created_at >= ?
    GROUP BY DATE(created_at)
    ORDER BY day ASC
  `, ["paid", new Date(Date.now() - 30 * 864e5).toISOString()]);
}
```

## Top-N (best-selling products)

```ts
interface TopProduct { name: string; sold: number; }

export async function topProducts(limit = 10) {
  return db.table<TopProduct>("order_items")
    .join("products", "order_items.product_id", "=", "products.id")
    .selectRaw(["products.name AS name", "SUM(order_items.qty) AS sold"])
    .groupBy("products.name")
    .orderBy("sold", "DESC")
    .limit(limit)
    .get();
}
```

## A cheap existence check

`exists()` is a `SELECT 1 … LIMIT 1` — far cheaper than `count() > 0` when you
only need a yes/no.

```ts
export async function hasPendingRefunds(): Promise<boolean> {
  return db.table("refunds").where("status", "=", "pending").exists();
}
```

## Exporting a large table without running out of memory

`chunk()` streams rows in batches and never loads the whole table at once — ideal
for CSV exports, re-indexing or backfills.

```ts
import { createWriteStream } from "node:fs";

export async function exportCustomersCsv(path: string) {
  const out = createWriteStream(path);
  out.write("id,email,created_at\n");

  let total = 0;
  await db.table<{ id: number; email: string; created_at: string }>("customers")
    .orderBy("id", "ASC")
    .chunk(1000, (rows) => {
      for (const c of rows) out.write(`${c.id},${c.email},${c.created_at}\n`);
      total += rows.length;
      // return false from here to stop early
    });

  out.end();
  return total;
}
```

::callout{type="info"}
`paginate(page, perPage)` is for UI pages (it returns totals and page metadata).
`chunk(size, cb)` is for processing **everything** with flat memory — pick the
one that matches the job.
::
