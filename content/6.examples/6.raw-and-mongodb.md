## Raw SQL

`raw()` is the escape hatch for anything the builder doesn't model. On SQL
engines, **always use `?` placeholders** — never interpolate values into the
string. (`examples/07-raw.js`)

```ts
// Parameterized query — values travel as bind params, not string concat
const rows = await db.raw(
  'SELECT name, age FROM users WHERE age > ? AND status = ? ORDER BY age DESC',
  [30, 'active']
);
```

### DDL and engine-specific SQL

```ts
// DDL
await db.raw('CREATE INDEX IF NOT EXISTS idx_users_status ON users(status)');

// Window functions, CTEs, anything your engine supports
const ranked = await db.raw(`
  SELECT name, balance,
         RANK() OVER (ORDER BY balance DESC) AS wealth_rank
  FROM users LIMIT 3
`);
```

::callout{type="warning"}
Bound parameters are not just for safety — they let the engine **reuse prepared
statements**. Interpolating values produces a unique SQL string every time and
defeats the cache (and opens you to SQL injection).
::

## MongoDB — same API, document engine

The **exact same builder API** works on MongoDB. Point a config entry at a
MongoDB instance and your code is unchanged. (`examples/09-mongodb.js`)

```ts
const db = dbcube.database('mongo_app');

// MongoDB has no autoincrement — provide ids yourself if you need them
await db.table('users').insert([
  { id: 1, name: 'Ada',   age: 36, status: 'active' },
  { id: 2, name: 'Linus', age: 54, status: 'active' },
]);

// where + orderBy — identical to SQL
const old = await db.table('users').where('age', '>', 50).orderBy('age', 'DESC').get();

// LIKE works (translated to an anchored, case-insensitive regex)
const ada = await db.table('users').where('name', 'LIKE', 'ad%').first();

// Atomic $inc
await db.table('users').where('id', '=', 1).increment('age', 1);

// upsert → native update command with upsert:true
await db.table('settings').upsert([{ key: 'theme', value: 'dark' }], ['key']);

// Aggregations
await db.table('users').count();
await db.table('users').avg('age');
```

### Transactions on MongoDB

Require a replica set (a single-node one is enough):

```bash
docker run -d --name mongo-rs -p 27017:27017 mongo:7 mongod --replSet rs0
docker exec mongo-rs mongosh --quiet --eval "rs.initiate()"
```

```ts
await db.transaction(async (trx) => {
  await trx.table('users').where('id', '=', 1).update({ status: 'tx' });
  await trx.table('users').where('id', '=', 2).update({ status: 'tx' });
});
```

### Raw command documents

On MongoDB, `raw()` takes a JSON command document (`db.runCommand`):

```ts
const res = await db.raw(JSON.stringify({
  find: 'users',
  filter: { age: { $gt: 30 } },
  sort: { age: -1 }
}));
console.log(res[0].cursor.firstBatch);
```

::callout{type="info"}
The same `where`, `orderBy`, `increment`, `upsert`, aggregations and transactions
you use on SQL engines translate to native MongoDB operations under the hood —
**write once, run on any engine**.
::
