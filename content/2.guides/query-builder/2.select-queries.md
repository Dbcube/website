## Basic Selection

### Select All Columns

Calling `get()` without `select()` fetches every column:

```typescript
const users = await db.table('users').get();
// SELECT * FROM users
```

### Select Specific Columns

`select()` receives an **array of column names**:

```typescript
const users = await db.table('users')
  .select(['id', 'name', 'email'])
  .get();
// SELECT id, name, email FROM users
```

Passing an empty array (or nothing) selects all columns:

```typescript
const users = await db.table('users').select().get();
// SELECT * FROM users
```

::callout{type="info"}
**Always prefer explicit columns** in production code: less data over the wire, stable payloads when the table grows, and self-documenting queries.
::

### Qualified Columns in JOINs

When joining tables, qualify columns with the table name to avoid ambiguity:

```typescript
const orders = await db.table('orders')
  .select(['orders.id', 'orders.total', 'users.name'])
  .join('users', 'orders.user_id', '=', 'users.id')
  .get();
```

## Distinct Results

`distinct()` removes duplicate rows from the result:

```typescript
const countries = await db.table('users')
  .distinct()
  .select(['country'])
  .get();
// SELECT DISTINCT country FROM users
```

## Selecting Computed Fields

If the connection was created with `useComputes()`, virtual columns defined with `@compute` in your `.cube` files can be selected like any real column:

```typescript
const db = await new Database('myapp').useComputes();

const users = await db.table('users')
  .select(['id', 'full_name'])   // full_name is computed from name + last_name
  .get();
```

Dbcube fetches the real dependency columns automatically and removes them from the result. See [Computed Fields](/guides/query-builder/computed-fields).

## Inspecting Available Columns

Use `columns()` to discover the table structure at runtime:

```typescript
const cols = await db.table('users').columns();
// ['id', 'name', 'email', 'created_at']
```

Useful for building dynamic admin panels or export tools:

```typescript
const cols = await db.table('products').columns();
const safeCols = cols.filter(c => c !== 'internal_cost');

const products = await db.table('products').select(safeCols).get();
```

## Combining with the Rest of the Builder

`select()` composes with every other method:

```typescript
const topCustomers = await db.table('users')
  .select(['users.id', 'users.name'])
  .join('orders', 'users.id', '=', 'orders.user_id')
  .where('orders.status', '=', 'paid')
  .groupBy('users.id')
  .orderBy('users.name', 'ASC')
  .limit(10)
  .get();
```

::callout{type="warning"}
**Aggregations are their own methods.** To count or sum, don't put `COUNT(...)` inside `select()` — use the dedicated executors: `await db.table('users').count()`, `.sum('amount')`, `.avg('price')`, `.max('created_at')`, `.min('price')`. See [Data Retrieval](/guides/query-builder/data-retrieval).
::

## Practical Examples

### Listing for a Dropdown

```typescript
const options = await db.table('categories')
  .select(['id', 'name'])
  .orderBy('name', 'ASC')
  .get();
```

### Unique Values for Filters

```typescript
const brands = await db.table('products')
  .distinct()
  .select(['brand'])
  .whereNotNull('brand')
  .orderBy('brand', 'ASC')
  .get();
```

### Lightweight Existence Check

```typescript
const found = await db.table('users')
  .select(['id'])
  .where('email', '=', 'john@example.com')
  .first();

const exists = found !== null;
```
