## Prerequisites

Before installing Dbcube, ensure you have:

- **Node.js** 16.0.0 or higher
- **npm**, **yarn**, or **pnpm** package manager
- A supported database (MySQL, PostgreSQL, SQLite, or MongoDB)

::callout{type="info"}
**TypeScript Support**: Dbcube is built with TypeScript and provides full type definitions out of the box. No additional `@types` packages are needed.
::

## Quick Installation

### Install Dbcube

Install the main Dbcube package in your project:

::code-group

```bash [npm]
npm install dbcube
```

```bash [yarn]
yarn add dbcube
```

```bash [pnpm]
pnpm add dbcube
```

```bash [bun]
bun add dbcube
```

::

This installs the core Dbcube query builder with full ORM functionality.

### Install CLI Tools (Optional)

The Dbcube CLI provides powerful database management commands:

::code-group

```bash [npm]
npm install -g @dbcube/cli
```

```bash [yarn]
yarn global add @dbcube/cli
```

```bash [pnpm]
pnpm add -g @dbcube/cli
```

```bash [bun]
pnpm add -g @dbcube/cli
```

::

::callout{type="warning"}
**Global Installation**: The CLI is typically installed globally so you can use `dbcube` commands from anywhere. You can also install it locally as a dev dependency if preferred.
::

### Install VS Code Extension

For the best development experience, install the Dbcube VS Code extension:

1. Open VS Code
2. Go to Extensions (`Ctrl+Shift+X`)
3. Search for "Dbcube"
4. Click Install

The extension provides:

- Syntax highlighting for `.cube` files
- IntelliSense and auto-completion
- Schema validation and error detection
- Code snippets and templates

## Database Drivers

Dbcube automatically includes drivers for supported databases. No additional installation is required for:

- **MySQL** - Built-in MySQL driver
- **PostgreSQL** - Built-in PostgreSQL driver
- **SQLite** - Built-in SQLite driver
- **MongoDB** - Built-in MongoDB driver

## Verify Installation

After installation, verify everything is working:

### Check ORM Installation

Create a simple test file:

```typescript
// test-dbcube.ts
import Database from "@dbcube/query-builder";

const db = new Database("myapp");
console.log("Dbcube Query Builder loaded successfully!");
```

### Check CLI Installation

Verify the CLI is installed correctly:

```bash
dbcube --version
```

You should see output similar to:

```
@dbcube/cli v5.1.4
```

### List Available Commands

See all available CLI commands:

```bash
dbcube --help
```

## Project Initialization

Once Dbcube is installed, create the configuration file:

### Create Configuration File

The easiest way is the interactive wizard, which generates `dbcube.config.js` and the `.env` variables for you:

```bash
npx dbcube run database:create
```

Or create `dbcube.config.js` in your project root manually:

```typescript
// dbcube.config.js
require("dotenv").config({ quiet: true });

module.exports = function (config) {
  config.set({
    databases: {
      myapp: {
        type: "mysql", // mysql, postgres, sqlite, mongodb
        config: {
          HOST: process.env.DBCUBE_MYAPP_HOST,
          USER: process.env.DBCUBE_MYAPP_USER,
          PASSWORD: process.env.DBCUBE_MYAPP_PASSWORD,
          DATABASE: process.env.DBCUBE_MYAPP_DATABASE,
          PORT: parseInt(process.env.DBCUBE_MYAPP_PORT, 10),
        },
      },
    },
  });
};
```

See the [Configuration guide](/getting-started/configuration) for all engines and options.

### Create Schema Directory

The easiest way is `npx dbcube init`, which scaffolds the whole `dbcube/`
directory for you. It organizes files by type under `dbcube/schema/`:

- `dbcube/schema/tables/` — `*.table.cube` table schema definitions
- `dbcube/schema/seeders/` — `*.seeder.cube` data seeders
- `dbcube/schema/triggers/` — `*.trigger.cube` database triggers
- `dbcube/schema/alters/` — `*.alter.cube` tracked migrations
- `dbcube/types/` — generated TypeScript types

The engine scans `dbcube/` **recursively**, so the subfolders are just for
organization — a flat `dbcube/*.cube` layout works too.

## Development Setup

For the best development experience:

### 1. TypeScript Configuration

If using TypeScript, ensure your `tsconfig.json` includes:

```json
{
  "compilerOptions": {
    "target": "ES2020",
    "module": "CommonJS",
    "moduleResolution": "node",
    "esModuleInterop": true,
    "allowSyntheticDefaultImports": true,
    "strict": true,
    "skipLibCheck": true
  }
}
```

### 2. Environment Variables

Create a `.env` file for sensitive configuration:

```bash
# .env
DBCUBE_MYAPP_HOST=localhost
DBCUBE_MYAPP_PORT=3306
DBCUBE_MYAPP_DATABASE=myapp_db
DBCUBE_MYAPP_USER=root
DBCUBE_MYAPP_PASSWORD=password
```

Then use in your config:

```typescript
// dbcube.config.js
require("dotenv").config({ quiet: true });

module.exports = function (config) {
  config.set({
    databases: {
      myapp: {
        type: "mysql",
        config: {
          HOST: process.env.DBCUBE_MYAPP_HOST,
          USER: process.env.DBCUBE_MYAPP_USER,
          PASSWORD: process.env.DBCUBE_MYAPP_PASSWORD,
          DATABASE: process.env.DBCUBE_MYAPP_DATABASE,
          PORT: parseInt(process.env.DBCUBE_MYAPP_PORT, 10),
        },
      },
    },
  });
};
```

### 3. Git Configuration

Add to your `.gitignore`:

```bash
# Dbcube local state and generated files
.dbcube/
dbcube/triggers/
dbcube/logs/

# Environment variables
.env
.env.local
.env.production
```

## Troubleshooting

### Installation Issues

**Permission errors on global install:**

```bash
# Use npx instead of global install
npx @dbcube/cli --version

# Or fix npm permissions
npm config set prefix ~/.npm-global
export PATH=~/.npm-global/bin:$PATH
```

**TypeScript import errors:**

```typescript
// Use require for CommonJS projects
const Database = require("@dbcube/query-builder").default;

// Use import for ES modules
import Database from "@dbcube/query-builder";
```

**Version conflicts:**

```bash
# Clear npm cache
npm cache clean --force

# Reinstall
npm uninstall @dbcube/query-builder
npm install @dbcube/query-builder
```

::callout{type="info"}
**Need more help?** Check our [full troubleshooting guide](/getting-started/troubleshooting) or visit our [GitHub repository](https://github.com/Dbcube/query-builder) for support.
::
