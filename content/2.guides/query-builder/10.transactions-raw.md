## Transactions

`db.transaction()` runs a callback inside a real database transaction. Everything executed through the received connection commits **atomically** — if the callback throws, the whole transaction rolls back automatically.

```typescript
import Database from '@dbcube/query-builder';

const db = new Database('myapp');

await db.transaction(async (trx) => {
  // Every operation through `trx` belongs to the same transaction
  await trx.table('accounts')
    .where('id', '=', 1)
    .update({ balance: 50 });

  await trx.table('accounts')
    .where('id', '=', 2)
    .update({ balance: 150 });

  await trx.table('transfers').insert([
    { from_id: 1, to_id: 2, amount: 100 }
  ]);
});
// All three operations committed together
```

### Automatic Rollback

Any error inside the callback aborts everything:

```typescript
try {
  await db.transaction(async (trx) => {
    await trx.table('orders').insert([{ user_id: 1, total: 99.9 }]);

    const stock = await trx.table('products').find(productId);
    if (stock.qty < 1) {
      throw new Error('Out of stock');   // ← triggers ROLLBACK
    }

    await trx.table('products').where('id', '=', productId).decrement('qty', 1);
  });
} catch (error) {
  // Nothing was written — the insert was rolled back too
}
```

### Return Values

The transaction resolves with whatever your callback returns:

```typescript
const order = await db.transaction(async (trx) => {
  const inserted = await trx.table('orders').insert([{ user_id: 1, total: 50 }]);
  return inserted[0];
});
```

### Rules and Behavior

- **Use `trx`, not `db`, inside the callback.** Operations on the outer `db` run *outside* the transaction.
- The transaction lives in the **Rust engine** (`BEGIN`/`COMMIT`/`ROLLBACK` on a dedicated connection), so isolation is the real database's isolation.
- Supported on **MySQL, PostgreSQL, and SQLite**. MongoDB standalone throws a clear error (Mongo transactions require a replica set).
- Requires an up-to-date query-engine binary — run `npx dbcube update` if you see a transaction support error.
- `raw()` also participates: `trx.raw('...')` executes inside the transaction.

## Raw SQL

`db.raw(query, params?)` is the escape hatch for anything the builder doesn't cover — always with **bound parameters**, never string interpolation:

```typescript
// Parameterized SELECT
const rows = await db.raw(
  'SELECT * FROM users WHERE age > ? AND status = ?',
  [25, 'active']
);

// DDL
await db.raw('CREATE INDEX idx_users_email ON users(email)');

// Engine-specific features
await db.raw('PRAGMA table_info(users)');                    // SQLite
await db.raw('EXPLAIN SELECT * FROM orders WHERE id = ?', [1]); // query plans
```

### Return Shape

- **Read queries** (SELECT, SHOW, PRAGMA, WITH, EXPLAIN, or anything with RETURNING) → array of row objects.
- **Write statements** → `[{ affectedRows, lastInsertId }]`.

```typescript
const result = await db.raw('UPDATE users SET status = ? WHERE id = ?', ['active', 5]);
// [{ affectedRows: 1, lastInsertId: 0 }]
```

### Parameter Binding

Parameters are bound by the Rust engine, not concatenated — SQL injection through values is impossible:

```typescript
// ✓ Safe — value is bound
await db.raw('SELECT * FROM users WHERE name = ?', [userInput]);

// ✗ NEVER do this
await db.raw(`SELECT * FROM users WHERE name = '${userInput}'`);
```

Use `?` placeholders for every engine — the engine translates to the native placeholder style internally.

### MongoDB

For MongoDB, `raw()` receives a **command document** (JSON string) instead of SQL:

```typescript
const stats = await db.raw('{"collStats": "users"}');
```

### Raw Inside Transactions

```typescript
await db.transaction(async (trx) => {
  await trx.raw('UPDATE counters SET value = value + 1 WHERE name = ?', ['orders']);
  await trx.table('orders').insert([{ user_id: 7, total: 12.5 }]);
});
```

## When to Use What

| Need | Use |
|------|-----|
| Normal CRUD and queries | The fluent builder |
| Multi-step writes that must succeed together | `transaction()` |
| Engine-specific SQL, DDL, window functions, CTEs | `raw()` |
| Per-group aggregations | `selectRaw()` + `groupBy()` + `having()` |
