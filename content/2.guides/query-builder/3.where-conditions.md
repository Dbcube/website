## Comparison Operators

`where(column, operator, value)` supports these operators:

| Operator | Meaning | Example |
|----------|---------|---------|
| `=` | Equal | `.where('status', '=', 'active')` |
| `!=` / `<>` | Not equal | `.where('role', '!=', 'admin')` |
| `>` `<` `>=` `<=` | Comparisons | `.where('age', '>=', 18)` |
| `LIKE` | Pattern match | `.where('name', 'LIKE', 'Jo%')` |
| `NOT LIKE` | Negated pattern | `.where('email', 'NOT LIKE', '%@spam.com')` |
| `IN` | In list | `.where('id', 'IN', [1, 2, 3])` |
| `NOT IN` | Not in list | `.where('status', 'NOT IN', ['banned', 'deleted'])` |
| `BETWEEN` | In range | `.where('age', 'BETWEEN', [18, 65])` |
| `NOT BETWEEN` | Outside range | `.where('price', 'NOT BETWEEN', [0, 10])` |
| `IS NULL` | Is null — **no value argument** | `.where('deleted_at', 'IS NULL')` |
| `IS NOT NULL` | Not null — **no value argument** | `.where('email', 'IS NOT NULL')` |

```typescript
const users = await db.table('users')
  .where('age', '>', 25)
  .get();

const noEmail = await db.table('users')
  .where('email', 'IS NULL')      // two arguments only
  .get();
```

::callout{type="info"}
The TypeScript overloads enforce this: `IS NULL` / `IS NOT NULL` accept exactly two arguments; every other operator requires a value.
::

## Combining Conditions

### AND (default)

Chained `where()` calls combine with AND:

```typescript
const results = await db.table('users')
  .where('status', '=', 'active')
  .where('age', '>=', 18)
  .get();
// WHERE status = 'active' AND age >= 18
```

### OR

```typescript
const results = await db.table('users')
  .where('role', '=', 'admin')
  .orWhere('role', '=', 'moderator')
  .get();
// WHERE role = 'admin' OR role = 'moderator'
```

### or() / and() Connectors

`or()` and `and()` set the connector used by the **next** condition:

```typescript
const results = await db.table('products')
  .where('stock', '>', 0)
  .or()
  .where('backorder', '=', true)
  .get();
// WHERE stock > 0 OR backorder = true
```

## Condition Groups

`whereGroup()` wraps conditions in parentheses — essential when mixing AND and OR:

```typescript
const users = await db.table('users')
  .where('status', '=', 'active')
  .whereGroup(query => {
    query.where('subscription', '=', 'premium')
         .orWhere('total_orders', '>', 100);
  })
  .get();
// WHERE status = 'active' AND (subscription = 'premium' OR total_orders > 100)
```

Groups receive a builder for the same table, so all condition methods work inside them.

## Convenience Methods

These helpers express common patterns more readably:

### whereBetween()

```typescript
const users = await db.table('users')
  .whereBetween('age', [20, 30])
  .get();
// WHERE age BETWEEN 20 AND 30
```

### whereIn()

```typescript
const users = await db.table('users')
  .whereIn('id', [1, 2, 3])
  .get();
// WHERE id IN (1, 2, 3)
```

### whereNotIn()

```typescript
const users = await db.table('users')
  .whereNotIn('status', ['banned', 'deleted'])
  .get();
// WHERE status NOT IN ('banned', 'deleted')
```

::callout{type="warning"}
`whereIn()` and `whereNotIn()` with an **empty array** add no condition (the call is ignored).
::

### whereNull() / whereNotNull()

```typescript
const unverified = await db.table('users')
  .whereNull('verified_at')
  .get();

const withPhone = await db.table('users')
  .whereNotNull('phone')
  .get();
```

## Text Search with LIKE

```typescript
const term = 'laptop';

const products = await db.table('products')
  .where('name', 'LIKE', `%${term}%`)
  .orWhere('description', 'LIKE', `%${term}%`)
  .get();
```

| Pattern | Matches |
|---------|---------|
| `'Jo%'` | Starts with "Jo" |
| `'%son'` | Ends with "son" |
| `'%oh%'` | Contains "oh" |

## Real-World Recipes

### Active, Adult, Recent Users

```typescript
const users = await db.table('users')
  .where('status', '=', 'active')
  .where('age', '>=', 18)
  .where('created_at', '>=', '2026-01-01')
  .orderBy('created_at', 'DESC')
  .get();
```

### Search with Optional Filters

```typescript
let query = db.table('products').where('published', '=', true);

if (filters.category) {
  query = query.where('category_id', '=', filters.category);
}
if (filters.maxPrice) {
  query = query.where('price', '<=', filters.maxPrice);
}
if (filters.search) {
  query = query.whereGroup(q => {
    q.where('name', 'LIKE', `%${filters.search}%`)
     .orWhere('sku', 'LIKE', `%${filters.search}%`);
  });
}

const products = await query.orderBy('name', 'ASC').get();
```

Because each call returns a new state, reassigning the variable is required (`query = query.where(...)`).

### Soft-Delete Awareness

```typescript
const visible = await db.table('posts')
  .whereNull('deleted_at')
  .where('status', '=', 'published')
  .get();
```
