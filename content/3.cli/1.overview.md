## Installation

Install the Dbcube CLI globally to use it from anywhere in your system:

::code-group

```bash [npm]
npm install -g @dbcube/cli
```

```bash [yarn]
yarn global add @dbcube/cli
```

```bash [pnpm]
pnpm add -g @dbcube/cli
```

```bash [bun]
bun add -g @dbcube/cli
```


::

Verify the installation:

```bash
dbcube --version
```

## Basic Usage

The CLI follows a simple command structure:

```bash
dbcube [command] [subcommand] [options]
```

### Getting Help

Get help for any command:

```bash
# General help
dbcube --help
dbcube -h

# Command-specific help
dbcube create --help
dbcube run --help
```

### Version Information

Check the CLI version:

```bash
dbcube --version
dbcube -v
```

## Available Commands

All Dbcube CLI commands use the `dbcube run` prefix:

### Database Management

```bash
# Create database from configuration
dbcube run database:create
```

### Schema Operations

```bash
# Create all tables (drops existing tables)
dbcube run table:fresh

# Update table structures (safer - keeps data)
dbcube run table:refresh
```

### Seeder Operations

```bash
# Run all seeders
dbcube run seeder:add
```

### Trigger Management

```bash
# Install/update all triggers
dbcube run trigger:fresh
```

### Binary Management

```bash
# Download database engine binaries
dbcube run download [engine] [version]

# Upgrade existing binaries
dbcube run upgrade
```

## Command Categories

### Database Commands

- `run database:create` - Create database from config file

### Schema Commands

- `run table:fresh` - Drop and recreate all tables
- `run table:refresh` - Update table structures without dropping

### Data Commands

- `run seeder:add` - Load all seeder files

### Trigger Commands

- `run trigger:fresh` - Install/update database triggers

### Binary Commands

- `run download` - Download database engine binaries
- `run upgrade` - Upgrade installed binaries

## Configuration

The CLI reads configuration from your `dbcube.config.js` file:

```typescript
// dbcube.config.js
module.exports = (dbcube) => {
  dbcube.addDatabase({
    name: "myapp",
    engine: "mysql",
    host: "localhost",
    port: 3306,
    database: "myapp_db",
    username: "root",
    password: "password",
  });
};
```

### Environment-Specific Configuration

Use different configurations for different environments:

```typescript
// dbcube.config.js
const env = process.env.NODE_ENV || "development";

module.exports = (dbcube) => {
  if (env === "development") {
    dbcube.addDatabase({
      name: "dev",
      engine: "mysql",
      host: "localhost",
      port: 3306,
      database: "myapp_dev",
      username: "root",
      password: "password",
    });
  } else if (env === "production") {
    dbcube.addDatabase({
      name: "prod",
      engine: "mysql",
      host: process.env.DB_HOST,
      port: process.env.DB_PORT,
      database: process.env.DB_NAME,
      username: process.env.DB_USER,
      password: process.env.DB_PASS,
    });
  }
};
```

## File Structure

The CLI works with this file structure:

```
project/
├── dbcube.config.js          # Main configuration
└── dbcube/                   # Dbcube directory
    ├── users.table.cube      # Table schemas
    ├── posts.table.cube
    ├── users.seeder.cube     # Data seeders
    ├── categories.seeder.cube
    ├── users.trigger.cube    # Database triggers
    └── audit.trigger.cube
```

You can organize files in subdirectories if preferred:

```
project/
├── dbcube.config.js
└── dbcube/
    ├── tables/
    │   ├── users.table.cube
    │   └── posts.table.cube
    ├── seeders/
    │   └── users.seeder.cube
    └── triggers/
        └── users.trigger.cube
```

## Common Workflows

### Setting Up a New Project

```bash
# 1. Create dbcube.config.js manually or copy from template

# 2. Create database
dbcube run database:create

# 3. Create all tables
dbcube run table:fresh

# 4. Seed initial data
dbcube run seeder:add

# 5. Install triggers
dbcube run trigger:fresh
```

### Development Workflow

```bash
# 1. Make schema changes in .cube files

# 2. Update tables (safer - keeps data)
dbcube run table:refresh

# 3. Update triggers if needed
dbcube run trigger:fresh

# 4. Add test data
dbcube run seeder:add
```

### Production Deployment

```bash
# 1. Fresh install on production
dbcube run database:create
dbcube run table:fresh

# 2. Install production data
dbcube run seeder:add

# 3. Install triggers
dbcube run trigger:fresh
```

## Error Handling

The CLI provides helpful error messages and suggestions:

```bash
$ dbcube run table:fresh
dbcube run table:refresh
❌ Error: Configuration file not found
💡 Suggestion: Run 'dbcube init' to create a configuration file

$ dbcube create database myapp
❌ Error: Database name 'myapp' already exists
💡 Suggestion: Use a different name or remove the existing configuration
```

## Troubleshooting

### Command Not Found

```bash
dbcube: command not found
```

**Solutions:**

- Reinstall CLI: `npm install -g @dbcube/cli`
- Check PATH: `echo $PATH`
- Use npx: `npx @dbcube/cli --help`

### Permission Errors

```bash
Error: EACCES: permission denied
```

**Solutions:**

- Run with sudo: `sudo dbcube run table:fresh
dbcube run table:refresh`
- Fix npm permissions: `npm config set prefix ~/.npm-global`
- Use local installation: `npx @dbcube/cli`

### Configuration Errors

```bash
Error: Invalid configuration
```

**Solutions:**

- Validate config file syntax
- Check database connection settings
- Run `dbcube init` to recreate configuration
