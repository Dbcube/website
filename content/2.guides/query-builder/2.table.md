`db.table('name')` returns a `Table` instance. Every builder method returns a **new immutable state**, so you can branch and reuse chains safely. This page is the complete reference — each method below exists in the current API.

## Basic Usage

```typescript
import Database from '@dbcube/query-builder';

const db = new Database('myapp');

const users = await db.table('users')
  .select(['id', 'name', 'email'])
  .where('status', '=', 'active')
  .orderBy('name', 'ASC')
  .limit(10)
  .get();
```

## Complete Method Reference

### Selection

| Method | Description |
|--------|-------------|
| `select(fields?: string[])` | Columns to fetch. Empty/omitted = all columns (`*`) |
| `distinct()` | Removes duplicate rows |
| `columns()` | Returns the table's column names (executes immediately) |

### Filtering

| Method | Description |
|--------|-------------|
| `where(col, op, value?)` | Adds an AND condition. Operators: `=`, `!=`, `<>`, `>`, `<`, `>=`, `<=`, `LIKE`, `NOT LIKE`, `IN`, `NOT IN`, `BETWEEN`, `NOT BETWEEN`, `IS NULL`, `IS NOT NULL` |
| `orWhere(col, op, value?)` | Adds an OR condition (same operators) |
| `whereGroup(callback)` | Parenthesized condition group |
| `or()` / `and()` | Sets the connector for the **next** condition |
| `whereBetween(col, [min, max])` | Range condition |
| `whereIn(col, values[])` | Membership condition |
| `whereNotIn(col, values[])` | Negated membership condition |
| `whereNull(col)` / `whereNotNull(col)` | NULL checks |

### Joins

| Method | Description |
|--------|-------------|
| `join(table, col1, op, col2)` | INNER JOIN |
| `leftJoin(table, col1, op, col2)` | LEFT JOIN |
| `rightJoin(table, col1, op, col2)` | RIGHT JOIN |

### Ordering, Grouping & Pagination

| Method | Description |
|--------|-------------|
| `orderBy(col, 'ASC' \| 'DESC')` | Sort results (chain multiple calls for multi-column sort) |
| `groupBy(col)` | Group rows |
| `having(col, op, value?)` | Filter grouped results |
| `selectRaw(expressions[])` | Add raw expressions to SELECT (`COUNT(*) AS n`) — pairs with `groupBy`/`having` |
| `limit(n)` | Maximum rows |
| `offset(n)` | Explicit row offset |
| `page(n)` | Pagination — requires `limit()`; offset = `(n - 1) * limit` |

### Relations

| Method | Description |
|--------|-------------|
| `with(relation, options?)` | [Eager-loads related records](/guides/query-builder/eager-loading) — one batched query, no N+1 |

### Execution — Reads

| Method | Returns | Description |
|--------|---------|-------------|
| `get()` | `DatabaseRecord[]` | All matching rows (resolves `with()` relations) |
| `first()` | `DatabaseRecord \| null` | First matching row |
| `find(value, column = 'id')` | `DatabaseRecord \| null` | Find by column value |
| `exists()` | `boolean` | Cheap existence check (`SELECT 1 ... LIMIT 1`) |
| `paginate(page, perPage)` | `PaginatedResult` | One call → `{ items, total, totalPages, hasNext, hasPrev }` |
| `chunk(size, callback)` | `void` | Processes all rows in batches; `return false` stops early |
| `count(column = '*')` | `number` | Count — **executes immediately** |
| `sum(column)` | `number` | Sum — executes immediately |
| `avg(column)` | `number` | Average — executes immediately |
| `max(column)` | `number` | Maximum — executes immediately |
| `min(column)` | `number` | Minimum — executes immediately |

### Execution — Writes

| Method | Requirement | Description |
|--------|-------------|-------------|
| `insert(data: object[])` | Array of objects | Inserts one or more rows |
| `update(data: object)` | At least one `where` | Updates matching rows |
| `delete()` | At least one `where` | Deletes matching rows |
| `upsert(rows, conflictKeys, updateCols?)` | Unique/PK on conflict keys | Insert-or-update in one statement |
| `increment(col, n = 1, extra?)` | At least one `where` | Atomic `SET col = col + n` |
| `decrement(col, n = 1, extra?)` | At least one `where` | Atomic `SET col = col - n` |
| `truncate()` | — | Deletes ALL rows (the only write allowed without WHERE — intent is explicit) |

::callout{type="warning"}
**Safety by design**: `update()`, `delete()`, `increment()`, and `decrement()` throw if you haven't added a WHERE condition — accidental full-table writes are impossible. To clear a table intentionally, `truncate()` says it loudly.
::

## Immutability and Reuse

Each method call clones the query state. The original chain is never mutated:

```typescript
const active = db.table('users').where('status', '=', 'active');

const firstPage = await active.orderBy('name', 'ASC').limit(20).get();
const activeCount = await active.count();   // unaffected by orderBy/limit above
```

## Aggregations Execute Immediately

Unlike `select()`, the aggregation methods **are** the execution — they return a `Promise<number>` directly:

```typescript
const total    = await db.table('users').count();
const adults   = await db.table('users').where('age', '>=', 18).count();
const revenue  = await db.table('orders').where('status', '=', 'paid').sum('amount');
const avgPrice = await db.table('products').avg('price');
const newest   = await db.table('posts').max('created_at');
const cheapest = await db.table('products').min('price');
```

```typescript
// ✗ Wrong — count() already executed, there is nothing to chain
const n = await db.table('users').count().first();

// ✓ Correct
const n = await db.table('users').count();
```

## Inspecting Table Structure

`columns()` returns the column names. For MongoDB it returns a hierarchical structure of nested documents:

```typescript
// SQL databases (MySQL, PostgreSQL, SQLite)
const cols = await db.table('users').columns();
// ['id', 'name', 'email', 'created_at']

// MongoDB
const structure = await db.table('users').columns();
// {
//   columns: ['id', 'name', 'profile'],
//   submenu: {
//     profile: { columns: ['age', 'city'], submenu: {} }
//   }
// }
```

## Full Example

```typescript
const report = await db.table('orders')
  .select(['orders.id', 'users.name', 'orders.total'])
  .join('users', 'orders.user_id', '=', 'users.id')
  .whereGroup(q => {
    q.where('orders.status', '=', 'paid')
     .orWhere('orders.status', '=', 'shipped');
  })
  .whereBetween('orders.total', [100, 5000])
  .whereNotNull('users.email')
  .orderBy('orders.total', 'DESC')
  .limit(25)
  .page(1)
  .get();
```
