Your `.table.cube` files already describe every table. `dbcube generate` turns them into TypeScript interfaces so your queries are typed end to end — like Prisma's generated client, but with zero runtime and your `.cube` files as the single source of truth.

## Generate

```bash
npx dbcube generate
```

```
  Generating types  2 tables

  users ..................................................... DONE
  sessions .................................................. DONE

  Types written to dbcube/types/index.ts
```

## What Gets Generated

For each table, **two interfaces** plus a schema map:

```typescript
// dbcube/types/index.ts (auto-generated — do not edit)

/** Table `users` · database `myapp` */
export interface User {
    id: number;
    name: string;
    last_name: string;
    /** computed field (read-only, requires useComputes()) */
    full_name: string | null;
}

/** Shape accepted by insert()/update() for `users` */
export interface NewUser {
    id?: number;          // optional: autoincrement
    name: string;         // required: not null, no default
    last_name: string;
}

/** Map of table name → row type. */
export interface DbSchema {
    users: User;
    sessions: Session;
}
```

The generator understands your schema semantics:

| Cube definition | Generated type |
|-----------------|----------------|
| `type: "int" / "decimal" / "float"` | `number` |
| `type: "varchar" / "text" / "date" / "timestamp"` | `string` |
| `type: "boolean"` | `boolean` |
| `type: "json"` | `any` |
| `type: "enum"; enumValues: ["a", "b"]` | `"a" \| "b"` (literal union) |
| without `"not null"` | `T \| null` |
| `autoincrement` / `defaultValue` | optional (`?`) in `New*` |
| `@compute` columns | included in the row type, **excluded** from `New*` (read-only) |

## Use the Types

```typescript
import Database from '@dbcube/query-builder';
import type { User, NewUser } from './dbcube/types';

const db = new Database('myapp');

// Typed reads
const users = await db.table('users').get() as User[];
const user = await db.table('users').find(1) as User | null;

// Typed inserts — the compiler enforces required fields
const newUser: NewUser = { name: 'Ana', last_name: 'García' };
await db.table('users').insert([newUser]);
```

## Keep Types in Sync

Types regenerate from the cubes, so the workflow is:

```bash
# 1. Edit your .table.cube
# 2. Apply the schema change
npx dbcube run table:refresh

# 3. Regenerate types
npx dbcube generate
```

Or let **dev mode** do both automatically on every save:

```bash
npx dbcube dev
```

::callout{type="info"}
If `dbcube/types/` exists, `dbcube dev` regenerates it automatically after each schema refresh — save the cube, get fresh types.
::

## CI Recommendation

Add generation + validation to your pipeline so schema drift fails the build:

```yaml
- run: npx dbcube validate          # .cube syntax / FK errors → exit 1
- run: npx dbcube generate          # refresh dbcube/types/
- run: git diff --exit-code dbcube/types/   # fail if types weren't committed
```
