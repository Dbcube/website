The `Database` class represents one configured database connection. Every query starts here: you create an instance with the **connection name** from your `dbcube.config.js` and call `table()` to begin building.

## Initial Setup

### Installation and Initialization

```typescript
import Database from '@dbcube/query-builder';

// The name must match a key under `databases` in dbcube.config.js
const db = new Database('myapp');

const users = await db.table('users').get();
```

Or through the ORM, which manages all instances for you:

```typescript
import { Dbcube } from 'dbcube';

const dbcube = new Dbcube();
dbcube.init();

const db = dbcube.database('myapp');
```

### Connection Configuration

The connection data lives in `dbcube.config.js` — the `Database` constructor only receives the name:

```javascript
// dbcube.config.js
require('dotenv').config({ quiet: true });

module.exports = function (config) {
  config.set({
    databases: {
      myapp: {
        type: "mysql",
        config: {
          HOST: process.env.DBCUBE_MYAPP_HOST,
          USER: process.env.DBCUBE_MYAPP_USER,
          PASSWORD: process.env.DBCUBE_MYAPP_PASSWORD,
          DATABASE: process.env.DBCUBE_MYAPP_DATABASE,
          PORT: parseInt(process.env.DBCUBE_MYAPP_PORT, 10)
        }
      }
    }
  });
};
```

For a **managed/cloud** database (Supabase, Neon, PlanetScale, MongoDB Atlas,
Turso), use a single `URL` connection string instead — TLS is enabled
automatically:

```javascript
myapp: {
  type: "postgres",
  config: { URL: process.env.DATABASE_URL }   // postgresql://...?sslmode=require
}
```

See the full format, including all cloud providers, in the
[Configuration guide](/getting-started/configuration#cloud-databases-url--tls).

## API Reference

The `Database` class exposes these methods:

| Method | Returns | Description |
|--------|---------|-------------|
| `table(name)` | `Table` | Starts a query builder chain for a table |
| `raw(sql, params?)` | `Promise<any>` | Executes [raw SQL with bound parameters](/guides/query-builder/transactions-raw) |
| `transaction(callback)` | `Promise<T>` | Runs the callback in an [atomic transaction](/guides/query-builder/transactions-raw) with auto-rollback |
| `connect()` | `Promise<void>` | Explicitly opens the connection via the query engine |
| `disconnect()` | `Promise<void>` | Closes the connection |
| `useComputes()` | `Promise<Database>` | Returns a new instance with [computed fields](/guides/query-builder/computed-fields) enabled |
| `useTriggers()` | `Promise<Database>` | Returns a new instance with [runtime triggers](/guides/query-builder/runtime-triggers) enabled |

### table()

Creates a `Table` instance — the object that holds the entire fluent API:

```typescript
// Select all records from a table
const users = await db.table('users').get();

// Select records with conditions
const activeUsers = await db.table('users')
  .where('status', '=', 'active')
  .orderBy('created_at', 'DESC')
  .limit(10)
  .get();

// Insert records (always an array)
await db.table('users').insert([
  { name: 'John', email: 'john@example.com', age: 30 }
]);

// Update records (WHERE is mandatory)
await db.table('users')
  .where('id', '=', 1)
  .update({ status: 'inactive' });

// Delete records (WHERE is mandatory)
await db.table('users')
  .where('status', '=', 'deleted')
  .delete();
```

### connect() / disconnect()

Connections are managed automatically per operation, but you can verify connectivity explicitly:

```typescript
const db = new Database('myapp');
await db.connect();      // throws a formatted error if the config is wrong
// ... work ...
await db.disconnect();
```

### useComputes() / useTriggers()

Both are **opt-in** features that return a *new* `Database` instance with the feature loaded from the local config database:

```typescript
const dbComputed = await new Database('myapp').useComputes();
const dbTriggered = await new Database('myapp').useTriggers();
```

Connections created without them have zero overhead from these features.

## Core Concepts

### Method Chaining

Builder methods (`select`, `where`, `orderBy`, …) return a new `Table` state, so chains are safe to split and reuse:

```typescript
const baseQuery = db.table('orders').where('status', '=', 'paid');

const recent = await baseQuery.orderBy('created_at', 'DESC').limit(10).get();
const total = await baseQuery.count();
```

### Return Types

| Operation | Returns |
|-----------|---------|
| `get()` | `DatabaseRecord[]` — array of plain objects |
| `first()`, `find()` | `DatabaseRecord \| null` |
| `count()`, `sum()`, `avg()`, `max()`, `min()` | `number` |
| `insert(data)` | The inserted records |
| `update(data)` | The update payload |
| `delete()` | Engine result |
| `columns()` | `string[]` (SQL) or hierarchical structure (MongoDB) |

## Error Handling

Errors come back with formatted output that includes the failing code location:

```typescript
try {
  const users = await db.table('users').where('age', '>', 25).get();
} catch (error) {
  // The console also shows:
  // [error] message from the engine
  // [code]  path/to/your/file.js:12:5 with a code excerpt
  // [help]  hint when available
  console.error('Query failed:', error.message);
}
```

Status codes from the engine: `200` success, `500` error, `600` warning.

## Best Practices

### 1. Always Use Try-Catch

```typescript
try {
  const users = await db.table('users').get();
} catch (error) {
  // handle or rethrow with context
}
```

### 2. Select Specific Columns

```typescript
// ✓ Only fetch what you need
const users = await db.table('users').select(['id', 'name']).get();

// ✗ Avoid SELECT * on wide tables
const users = await db.table('users').get();
```

### 3. Use Pagination for Large Datasets

```typescript
const pageSize = 50;
const page = 2;

const results = await db.table('logs')
  .orderBy('created_at', 'DESC')
  .limit(pageSize)
  .page(page)        // OFFSET is computed as (page - 1) * limit
  .get();
```

### 4. Reuse One Instance per Database

Create the `Database` once per connection name and share it — or let the [ORM](/guides/orm/overview) do this for you.
