## Quickstart

The smallest useful program: connect, read, filter, write.
(`examples/01-quickstart.js`)

```ts
import { dbcube } from "dbcube";

async function main() {
  // One Database instance per entry in dbcube.config.js
  const db = dbcube.database('demo');

  // Read everything
  const users = await db.table('users').get();
  console.log(`${users.length} users in the table`);

  // Filter + order + limit
  const active = await db.table('users')
    .where('status', '=', 'active')
    .orderBy('age', 'DESC')
    .limit(2)
    .get();

  // Insert (always an array — inserting many rows is one call)
  await db.table('users').insert([
    { name: 'Margaret Hamilton', email: 'margaret@example.com', age: 32 }
  ]);
}
main();
```

## Reading data

Every read pattern. (`examples/02-reads.js`)

### Selecting columns

```ts
const names = await db.table('users').select(['id', 'name']).get();
```

### WHERE conditions

```ts
// where + orWhere
const adaOrOld = await db.table('users')
  .where('name', 'LIKE', 'Ada%')
  .orWhere('age', '>', 80)
  .get();

// whereGroup → WHERE status='active' AND (age<40 OR age>80)
const grouped = await db.table('users')
  .where('status', '=', 'active')
  .whereGroup(q => { q.where('age', '<', 40).orWhere('age', '>', 80); })
  .get();

// set helpers
await db.table('users').whereIn('id', [1, 2]).get();
await db.table('users').whereNotIn('status', ['inactive']).get();
await db.table('users').whereBetween('age', [30, 60]).get();
await db.table('users').whereNull('deleted_at').get();
await db.table('users').whereNotNull('email').get();
```

### Single rows: `first` and `find`

```ts
const first  = await db.table('users').orderBy('age', 'ASC').first();
const byId   = await db.table('users').find(3);                       // by primary key
const byEmail = await db.table('users').find('linus@example.com', 'email'); // by any column
```

### Distinct, pagination, joins

```ts
// distinct
const statuses = await db.table('users').select(['status']).distinct().get();

// manual pagination
const page2 = await db.table('users').orderBy('id', 'ASC').limit(2).offset(2).get();

// joins (SQL engines)
const withOrders = await db.table('orders')
  .join('users', 'orders.user_id', '=', 'users.id')
  .select(['orders.product', 'users.name'])
  .get();
```

::callout{type="info"}
`get()` returns an array, `first()` / `find()` return a single object (or
`null`). Aggregations like `count()` return a number directly — see
[Aggregations](/examples/aggregations).
::

## Writing data

Insert, update, delete, upsert, counters and truncate. (`examples/03-writes.js`)

### Insert returns the rows

```ts
const inserted = await db.table('users').insert([
  { name: 'Katherine Johnson', email: 'katherine@example.com', age: 33 },
]);
console.log('new id:', inserted[0].id); // generated id/uuid returned
```

### Update requires a WHERE (safety)

```ts
// Throws — mass updates without a filter are blocked on purpose
await db.table('users').update({ status: 'x' });

// Correct: scoped update
await db.table('users').where('id', '=', 1).update({ status: 'verified' });
```

### Upsert

```ts
// Insert, or update on conflict (the key column must be UNIQUE)
await db.table('settings').upsert([{ key: 'theme', value: 'dark' }],  ['key']);
await db.table('settings').upsert([{ key: 'theme', value: 'light' }], ['key']);
// → a single row, value = 'light'
```

### Atomic counters

```ts
// No read-modify-write race — done in one SQL statement
await db.table('users').where('id', '=', 1).increment('balance', 250);
await db.table('users').where('id', '=', 2).decrement('balance', 100);

// Update extra columns in the same statement
await db.table('users').where('id', '=', 1).increment('balance', 1, { status: 'vip' });
```

### Delete and truncate

```ts
await db.table('users').where('email', '=', 'katherine@example.com').delete();

// truncate is the ONLY write allowed without a WHERE — explicit destructive intent
await db.table('settings').truncate();
```

::callout{type="warning"}
`update()` and `delete()` **require a `where()`** and throw without one. The only
exception is `truncate()`, which is intentionally explicit about wiping a table.
::
