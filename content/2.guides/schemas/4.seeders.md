## What are Seeders?

Seeders are files that define datasets to insert into specific tables:

- **Initial Data**: Basic configuration required by the application
- **Test Data**: Information for development and testing
- **Demo Content**: Examples to showcase functionality
- **Configurations**: Roles, permissions, predefined categories

## Basic Seeder Structure

::cube-code
---
code: |
    @database("database_name");        // Target database
    @table("table_name");              // Table to populate

    @fields("field1", "field2", "field3");  // Columns to insert

    @dataset(
        ("value1", "value2", "value3"),      // First record
        ("value4", "value5", "value6"),      // Second record
        ("value7", "value8", "value9")       // Third record
    );
---
::

### Basic Example: Users

::cube-code
---
code: |
    // users.seeder.cube
    @database("blog");
    @table("users");

    @fields("name", "email", "password", "status");

    @dataset(
        ("Main Admin", "admin@example.com", "$2b$12$hashedpassword1", "active"),
        ("John Doe", "john@example.com", "$2b$12$hashedpassword2", "active"),
        ("Jane Smith", "jane@example.com", "$2b$12$hashedpassword3", "active"),
        ("Bob Wilson", "bob@example.com", "$2b$12$hashedpassword4", "inactive")
    );
---
::

## Seeder Organization

### Execution Order

Dbcube automatically resolves dependencies based on your table relationships. You can also use numeric prefixes for explicit order control:

```
project/
└── dbcube/
    ├── 01-users.seeder.cube          # First: base tables
    ├── 02-roles.seeder.cube
    ├── 03-user-roles.seeder.cube     # After: tables with foreign keys
    ├── 04-categories.seeder.cube
    ├── 05-posts.seeder.cube          # Then: main content
    ├── 06-comments.seeder.cube       # Finally: dependent data
    └── 99-test-data.seeder.cube      # Last: testing data
```

### Environment-Specific Seeders

::cube-code
---

## Seeder Commands

### Running Seeders

```bash
# Run all seeders (loads all .seeder.cube files)
dbcube run seeder:add
```

The CLI will:
1. Find all `.seeder.cube` files in the `dbcube/` directory
2. Resolve dependencies between tables
3. Execute seeders in the correct order
4. Show progress for each seeder
5. Display a summary of successes and errors

## Best Practices

### 1. Logical Execution Order

```bash
# ✅ Follow dependencies
01-roles.seeder.cube              # No dependencies
02-users.seeder.cube              # Depends on roles
03-categories.seeder.cube         # No dependencies
04-products.seeder.cube           # Depends on categories
05-orders.seeder.cube             # Depends on users
06-order-items.seeder.cube        # Depends on orders and products
```

### 2. Consistent Data

::cube-code
---
code: |
    @database("blog");
    @table("posts");

    @fields("title", "slug", "user_id", "category_id", "status");

    @dataset(
        // ✅ Consistent IDs with other seeders
        ("Post 1", "post-1", 1, 1, "published"),  // user_id=1 must exist
        ("Post 2", "post-2", 2, 2, "draft"),      // category_id=2 must exist

        // ❌ Avoid IDs that don't exist in other tables
        ("Post 3", "post-3", 99, 1, "published")  // user_id=99 might not exist
    );
---
::

## Troubleshooting

### Common Errors

::cube-code
---
code: |
    // ❌ Error: Wrong number of fields
    @fields("name", "email");  // 2 fields defined

    @dataset(
        ("John", "john@email.com", "extra_value")  // 3 values - ERROR
    );

    // ✅ Solution: Match fields and values
    @fields("name", "email", "status");  // 3 fields

    @dataset(
        ("John", "john@email.com", "active")  // 3 values - CORRECT
    );
---
::

::cube-code
---
code: |
    // ❌ Error: Wrong data type
    @fields("price", "active");

    @dataset(
        ("not_a_number", "not_a_boolean")  // Wrong types
    );

    // ✅ Solution: Use correct types
    @fields("price", "active");

    @dataset(
        (99.99, true)  // Correct number and boolean
    );
---
::

::cube-code
---
code: |
    // ❌ Error: Invalid foreign key
    @fields("user_id", "name");

    @dataset(
        (999, "Post")  // user_id=999 probably doesn't exist
    );

    // ✅ Solution: Use existing IDs
    @fields("user_id", "name");

    @dataset(
        (1, "Post")  // Make sure user_id=1 exists
    );
---
::
