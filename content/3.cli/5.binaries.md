Dbcube's performance comes from three native **Rust engines** distributed as standalone binaries. The CLI downloads the right build for your OS and architecture automatically, and gives you full control over versions.

| Engine | Role |
|--------|------|
| `query-engine` | Executes DML (select/insert/update/delete) against MySQL, PostgreSQL, SQLite, MongoDB |
| `schema-engine` | Parses `.cube` files and executes DDL, seeders, and trigger registration |
| `sqlite-engine` | Dedicated SQLite executor |

## update

Checks installed binaries against the latest published versions and upgrades anything outdated.

```bash
npx dbcube update
```

```bash
$ npx dbcube update

Checking binary versions...
  query-engine   v3.1.0 → v3.1.1  (updating)
  schema-engine  v3.2.0           (up to date)
  sqlite-engine  v1.4.2           (up to date)

✓ query-engine updated to v3.1.1
```

Run this after upgrading the npm packages to keep engines and builders in sync.

## run download

Downloads engines explicitly — interactively, by name, or pinned to a version.

```bash
# Interactive: pick engine and version from a menu
npx dbcube run download

# Latest version of a specific engine
npx dbcube run download query-engine

# Pin an exact version
npx dbcube run download query-engine v3.1.1
```

Valid engine names: `query-engine`, `schema-engine`, `sqlite-engine`.

### When to Pin Versions

- **CI/CD reproducibility** — pin exact versions in your pipeline so builds never drift.
- **Rollback** — if a new engine version misbehaves, downgrade instantly without touching npm packages.
- **Air-gapped prep** — download binaries ahead of time on a connected machine.

## Platform Detection

Binaries are published per platform and selected automatically:

| Platform | Example artifact |
|----------|------------------|
| Windows x64 | `query-engine-windows-x64.exe` |
| Linux x64 | `query-engine-linux-x64` |
| macOS x64 | `query-engine-macos-x64` |
| macOS ARM64 (Apple Silicon) | `query-engine-macos-arm64` |

## Troubleshooting

### "Binary not initialized" / engine not found

Run a download to restore the binaries:

```bash
npx dbcube run download
```

### Process timeout

Engine calls time out after 30 seconds by default. Timeouts usually mean the database host is unreachable — verify the connection values in `dbcube.config.js` before suspecting the binary.

### Corporate proxies / no internet

Download the binaries on a machine with access using `run download`, then copy them to the same location on the target machine.
