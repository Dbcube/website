## database:create

Interactive wizard that configures **and** creates a database in two phases.

### Usage

```bash
npx dbcube run database:create
```

### What It Does

1. If `dbcube.config.js` doesn't exist, it is created automatically with an empty `databases` section.
2. If you already have configured databases, a menu lets you **pick an existing configuration** (to create its physical database) or **create a new configuration**.
3. **Phase 1 — Configuration**: prompts for the engine (`mysql`, `postgres`, `sqlite`, `mongodb`), the connection name, and the connection data. Then it:
   - Inserts the new entry into `dbcube.config.js` (using `config.set({ databases: ... })`)
   - Writes the credentials to your `.env` as `DBCUBE_<NAME>_*` variables
4. **Phase 2 — Physical creation**: the schema-engine connects with that configuration and creates the actual database.

### What Gets Generated

For a MySQL connection named `myapp`, the wizard produces:

```bash
# .env (created or updated)
DBCUBE_MYAPP_HOST=localhost
DBCUBE_MYAPP_PORT=3306
DBCUBE_MYAPP_DATABASE=myapp_db
DBCUBE_MYAPP_USER=root
DBCUBE_MYAPP_PASSWORD=password
```

```typescript
// dbcube.config.js (entry inserted automatically)
require("dotenv").config({ quiet: true });

module.exports = function (config) {
  config.set({
    databases: {
      myapp: {
        type: "mysql",
        config: {
          HOST: process.env.DBCUBE_MYAPP_HOST,
          USER: process.env.DBCUBE_MYAPP_USER,
          PASSWORD: process.env.DBCUBE_MYAPP_PASSWORD,
          DATABASE: process.env.DBCUBE_MYAPP_DATABASE,
          PORT: parseInt(process.env.DBCUBE_MYAPP_PORT, 10),
        },
      },
    },
  });
};
```

For SQLite, only the database name is needed:

```typescript
local: {
  type: "sqlite",
  config: {
    DATABASE: process.env.DBCUBE_LOCAL_DATABASE   // file at .dbcube/<name>.db
  }
}
```

::callout{type="info"}
**Secrets never land in the config file.** The wizard always writes real values to `.env` and references them from `dbcube.config.js` via `process.env`.
::

## database:create:config

Runs **only Phase 1**: adds a database configuration to `dbcube.config.js` and `.env`, without creating the physical database.

```bash
npx dbcube run database:create:config
```

Useful when the database already exists on the server and you just need Dbcube to know about it.

## database:create:physical

Runs **only Phase 2**: creates the physical database using a configuration that already exists in `dbcube.config.js`.

```bash
npx dbcube run database:create:physical
```

Useful in deployment scripts where the configuration is committed (pointing at env vars) and you only need to provision the database.

## Verify the Result

```bash
# MySQL
mysql -u root -p -e "SHOW DATABASES LIKE 'myapp_db';"

# PostgreSQL
psql -U postgres -l | grep myapp_db

# SQLite — file created inside your project
ls .dbcube/
```

## Troubleshooting

### Database Already Exists

Selecting an existing configuration re-runs the physical creation against it — if the database already exists, the engine reports it and continues without destroying anything.

### Connection Errors

**Error: Can't connect to database server**

```bash
# Check if the database server is running
sudo systemctl status mysql       # MySQL
sudo systemctl status postgresql  # PostgreSQL
```

**Error: Access denied**

Verify the credentials written to `.env`, and that the user can create databases:

```sql
-- MySQL
GRANT CREATE ON *.* TO 'username'@'localhost';
FLUSH PRIVILEGES;

-- PostgreSQL
ALTER USER username CREATEDB;
```

### Config File Issues

If the wizard can't find the `databases: {` section, your `dbcube.config.js` was probably edited into a different shape. The expected skeleton is:

```typescript
require("dotenv").config({ quiet: true });

module.exports = function (config) {
  config.set({
    databases: {},
  });
};
```
