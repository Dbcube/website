## The short version

Dbcube runs query parsing, SQL generation, execution and row decoding in a
**native Rust engine** — off the Node.js event loop. In a head-to-head against
the four most popular Node ORMs, on a real PostgreSQL 16 database, Dbcube **beats
Prisma in every operation**, and is the *fastest library in the table* for reads,
transactions and concurrent traffic. `🏆` = #1 of all five; `✅` = faster than
Prisma.

| Operation | Dbcube | Prisma | Verdict |
|---|---:|---:|---|
| SELECT by primary key | **1.15 ms** | 1.85 ms | 🏆 −38% · #1 |
| Filtered list (WHERE+ORDER+LIMIT) | **1.10 ms** | 1.53 ms | 🏆 −28% · #1 |
| COUNT with filter | **0.83 ms** | 1.09 ms | 🏆 −24% · #1 |
| Relation load (50 users + orders) | 4.00 ms | 5.31 ms | ✅ −25% (Knex #1) |
| INSERT single row | 1.64 ms | 2.00 ms | ✅ −18% (TypeORM #1) |
| UPDATE by primary key | 1.69 ms | 2.44 ms | ✅ −31% (Knex #1) |
| Transaction (2 atomic updates) | **2.15 ms** | 4.70 ms | 🏆 −54% · #1 |
| Bulk INSERT 1,000 rows | 16.48 ms | 23.63 ms | ✅ −30% (Knex #1) |
| 100 concurrent PK lookups (per round) | **8.29 ms** | 9.35 ms | 🏆 −11% · #1 |

## Why it's fast

- **Native execution** — query parsing, SQL generation, execution and row
  decoding happen in compiled Rust, keeping your Node.js event loop free.
- **Built for concurrency** — parallel queries scale cleanly, so real traffic
  (many requests at once) stays fast instead of serializing.
- **Single-round-trip transactions** — `db.transaction()` / `db.batch()` commit
  a whole sequence of writes in one network round-trip.

## Where Dbcube shines (and where it doesn't)

The honest read:

- **Reads, updates, transactions, bulk, concurrency** — Dbcube beats Prisma
  consistently and is at or near the top of the whole field.
- **Raw bulk throughput** — a thin query builder like **Knex** still edges out
  everyone on a 1,000-row insert (it's almost the raw driver). Dbcube beats
  Prisma and Drizzle here, but Knex is the raw-speed champion.
- **Single inserts** — Dbcube, Drizzle and Knex are within noise of each other;
  Dbcube comfortably beats Prisma.

This is a benchmark you can audit line-by-line: every contender's code is ~80
lines in `benchmarks/src/contenders/*.js`, the schema and seed data are
identical, and the connection budget is capped at 10 for everyone.
