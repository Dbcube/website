## What is the Dbcube VSCode Extension?

The **Dbcube VSCode Extension** (officially named `cube-lang`) provides rich language support for `.cube` files in Visual Studio Code. It transforms the experience of writing database schemas from plain text editing to a fully-featured IDE experience.

## Installation

### From VS Code Marketplace

1. Open Visual Studio Code
2. Open the Extensions view (Ctrl+Shift+X or Cmd+Shift+X)
3. Search for "Dbcube" or "cube-lang"
4. Click **Install**

### From Command Line

```bash
code --install-extension Dbcube.cube-lang
```

### Manual Installation

1. Download the `.vsix` file from the [releases page](https://github.com/Dbcube/vscode-extension-formater/releases)
2. Open VS Code
3. Go to Extensions view
4. Click the `...` menu
5. Select "Install from VSIX..."
6. Choose the downloaded file

## Features

### Syntax Highlighting

The extension provides rich syntax highlighting for all `.cube` file types:

#### Table Files (`.table.cube`)

::cube-code
---
code: |
  @database("myapp");

  @meta({
    name: "users";
    description: "User accounts";
  });

  @columns({
    id: {
      type: "int";
      options: ["primary", "autoincrement"];
    };
    email: {
      type: "varchar";
      length: "255";
      options: ["not null", "unique"];
    };
  });
---
::

- **Annotations** (`@database`, `@meta`, `@columns`) - Highlighted in blue
- **Keywords** (`type`, `options`, `length`) - Highlighted in purple
- **Strings** - Highlighted in green
- **Numbers** - Highlighted in orange
- **Data types** (`int`, `varchar`, `timestamp`) - Highlighted distinctly

#### Seeder Files (`.seeder.cube`)

::cube-code
---
code: |
  @database("myapp");
  @table("users");

  @fields("name", "email", "status");

  @dataset(
    ("John Doe", "john@example.com", "active"),
    ("Jane Smith", "jane@example.com", "active")
  );
---
::

- **Annotations** - Clear visual distinction
- **String values** - Properly highlighted
- **Field lists** - Easy to read

#### Trigger Files (`.trigger.cube`)

::cube-code
---
code: |
  @database("myapp");
  @table("users");

  @beforeAdd({
    name: "validate_email";
    function: ({db, newData}) => {
      if (!newData.email.includes('@')) {
        throw new Error('Invalid email');
      }
    };
  });
---
::

- **typescript code** - Full syntax highlighting inside functions
- **Trigger types** - Clearly marked
- **Function parameters** - Properly highlighted

#### Column Options

### Real-time Validation

The extension validates your `.cube` files as you type:

#### Syntax Errors

::cube-code
---
code: |
  @database("myapp");

  @meta({
    name: "users"  // ❌ Error: Missing semicolon
  });
---
::

Red underline appears with error message: "Expected semicolon"

#### Database Configuration

::cube-code
---
code: |
  @database("nonexistent");  // ⚠️ Warning if database not in config
---
::

Yellow underline with warning: "Database 'nonexistent' not found in dbcube.config.js"

#### Type Validation

::cube-code
---
code: |
  @columns({
    age: {
      type: "integer";  // ❌ Error: Invalid type
      options: ["primary", "autoincrement"];
    };
  });
---
::

Error message: "Invalid data type 'integer'. Did you mean 'int'?"

### Commands

Access extension commands via Command Palette (Ctrl+Shift+P or Cmd+Shift+P):

#### Validate Cube File

- **Command**: `Dbcube: Validate Cube File`
- **Shortcut**: None (can be configured)
- **Description**: Validates the current `.cube` file for syntax and semantic errors

#### Format Cube Document

- **Command**: `Dbcube: Format Cube Document`
- **Shortcut**: Shift+Alt+F (default format shortcut)
- **Description**: Formats the current `.cube` file with proper indentation and spacing

Example before formatting:

::cube-code
---
code: |
  @database("myapp");@table("users");@fields("name","email");
---
::

After formatting:
::cube-code
---
code: |
  @database("myapp");
  @table("users");

  @fields("name", "email");
---
::

## File Associations

The extension automatically recognizes these file patterns:

- `*.cube` - Generic cube files
- `*.table.cube` - Table schema definitions
- `*.seeder.cube` - Data seeder files
- `*.trigger.cube` - Trigger definition files

No manual configuration needed!

## Configuration

### Extension Settings

Access via File > Preferences > Settings, then search for "Dbcube":

#### `dbcube.validation.enabled`

- **Type**: Boolean
- **Default**: `true`
- **Description**: Enable/disable real-time validation

```json
{
  "dbcube.validation.enabled": true
}
```

#### `dbcube.format.indentSize`

- **Type**: Number
- **Default**: `2`
- **Description**: Number of spaces for indentation

```json
{
  "dbcube.format.indentSize": 2
}
```

#### `dbcube.format.insertSpaces`

- **Type**: Boolean
- **Default**: `true`
- **Description**: Use spaces instead of tabs

```json
{
  "dbcube.format.insertSpaces": true
}
```

#### `dbcube.diagnostics.level`

- **Type**: String
- **Default**: `"error"`
- **Options**: `"error"`, `"warning"`, `"info"`, `"off"`
- **Description**: Diagnostic level for validation messages

```json
{
  "dbcube.diagnostics.level": "warning"
}
```

### Workspace Settings

Configure settings per project in `.vscode/settings.json`:

```json
{
  "dbcube.validation.enabled": true,
  "dbcube.format.indentSize": 4,
  "dbcube.diagnostics.level": "error",
  "[cube]": {
    "editor.formatOnSave": true,
    "editor.defaultFormatter": "Dbcube.cube-lang"
  }
}
```

## Usage Examples

### Creating a New Table Schema

1. Create a new file: `users.table.cube`
2. Type `@database` - autocomplete suggests the annotation
3. Add your database name from config
4. Type `@meta` - get structure suggestion
5. Type `@columns` - autocomplete helps with column definitions
6. Save file - validation runs automatically

### Formatting on Save

Enable auto-format in VS Code settings:

```json
{
  "editor.formatOnSave": true,
  "editor.defaultFormatter": "Dbcube.cube-lang"
}
```

Now every time you save a `.cube` file, it's automatically formatted.

### Working with Large Schema Files

The extension handles large files efficiently:

- ✓ Syntax highlighting for files with 1000+ lines
- ✓ Validation runs in background (non-blocking)
- ✓ IntelliSense remains responsive
- ✓ Format command optimized for large documents

## Keyboard Shortcuts

Default shortcuts for cube files:

| Action | Windows/Linux | macOS |
|--------|---------------|-------|
| Format Document | Shift+Alt+F | Shift+Option+F |
| Validate File | None (use Command Palette) | None (use Command Palette) |
| Go to Definition | F12 | F12 |
| Find All References | Shift+F12 | Shift+F12 |

### Custom Shortcuts

Add custom shortcuts in `keybindings.json`:

```json
[
  {
    "key": "ctrl+shift+v",
    "command": "dbcube.validateFile",
    "when": "editorLangId == cube"
  },
  {
    "key": "ctrl+shift+f",
    "command": "editor.action.formatDocument",
    "when": "editorLangId == cube"
  }
]
```

## Integration with CLI

The VSCode extension works seamlessly with the Dbcube CLI:

### Workflow

1. **Edit** `.cube` files in VS Code (with syntax highlighting and validation)
2. **Save** files (auto-formatted)
3. **Run** CLI commands in integrated terminal:
   ```bash
   dbcube run table:fresh
   dbcube run seeder:add
   dbcube run trigger:fresh
   ```
4. **See results** in terminal output
5. **Repeat** as needed

### Terminal Integration

The extension adds Dbcube commands to the terminal context menu:

1. Right-click in a `.cube` file
2. Select "Run in Terminal"
3. Choose command:
   - Create/refresh tables
   - Run seeders
   - Install triggers

## Troubleshooting

### Extension Not Working

**Problem**: Syntax highlighting not showing

**Solutions**:
1. Check file extension is `.cube`, `.table.cube`, `.seeder.cube`, or `.trigger.cube`
2. Reload VS Code: Press Ctrl+Shift+P → "Developer: Reload Window"
3. Verify extension is enabled: Extensions view → Search "Dbcube"
4. Check language mode (bottom right of editor) - should show "Cube"

### Validation Errors Not Showing

**Problem**: No error underlines appear

**Solutions**:
1. Check `dbcube.validation.enabled` is `true`
2. Verify `dbcube.config.js` exists in project root
3. Check Output panel: View → Output → Select "Dbcube Language Server"
4. Reload window

### Autocomplete Not Working

**Problem**: No suggestions when typing

**Solutions**:
1. Wait a moment after opening file (language server initializing)
2. Try manually: Ctrl+Space (Windows/Linux) or Cmd+Space (macOS)
3. Check VS Code version (minimum v1.60.0 required)
4. Disable conflicting extensions

### Format Command Not Available

**Problem**: "Format Document" option missing

**Solutions**:
1. Ensure file is saved with `.cube` extension
2. Check default formatter: Right-click → "Format Document With..." → Choose "Dbcube"
3. Update extension to latest version

## Best Practices

### File Organization

Organize `.cube` files logically for better IDE experience:

```
dbcube/
├── tables/
│   ├── core/
│   │   ├── users.table.cube
│   │   └── roles.table.cube
│   └── features/
│       └── posts.table.cube
├── seeders/
│   └── 01-users.seeder.cube
└── triggers/
    └── audit.trigger.cube
```

### Use Workspace

Open the project folder (not individual files) for best results:
- File → Open Folder
- Select your project root directory

This enables:
- Proper config file detection
- Database name validation
- Better IntelliSense suggestions

### Enable All Features

Recommended `settings.json` configuration:

```json
{
  "dbcube.validation.enabled": true,
  "dbcube.format.indentSize": 2,
  "dbcube.diagnostics.level": "warning",
  "[cube]": {
    "editor.formatOnSave": true,
    "editor.defaultFormatter": "Dbcube.cube-lang",
    "editor.suggest.showWords": false,
    "editor.quickSuggestions": {
      "other": true,
      "comments": false,
      "strings": true
    }
  }
}
```
