`.with()` eager-loads related rows with **one batched query per relation** — no
N+1. Relations resolve automatically from the `foreign` definitions in your
`.cube` files, or explicitly via options. (`examples/06-relations.js`)

## hasMany (auto-detected)

```ts
// orders.user_id → users.id, detected from the .cube foreign key
const users = await db.table('users').with('orders').orderBy('id', 'ASC').get();

for (const u of users) {
  console.log(`${u.name}: ${u.orders.length} order(s)`, u.orders.map(o => o.product));
}
```

Each user gets an `orders` array. Internally Dbcube runs **two** queries (one for
users, one `WHERE user_id IN (...)` for all their orders) and stitches them
together — never one query per user.

## belongsTo (explicit options)

Attach the parent as a single object:

```ts
const orders = await db.table('orders')
  .with('buyer', { table: 'users', foreignKey: 'user_id', type: 'one' })
  .get();

for (const o of orders) {
  console.log(`"${o.product}" was bought by ${o.buyer?.name}`);
}
```

| Option | Meaning |
|---|---|
| `table` | The related table |
| `foreignKey` | The column linking the two tables |
| `type: 'one'` | Attach a single object (belongsTo / hasOne) |
| `type: 'many'` | Attach an array (hasMany) — the default |

## Relations compose with everything

`.with()` chains with any other clause:

```ts
const richBuyers = await db.table('users')
  .where('balance', '>', 400)
  .with('orders')
  .get();
```

::callout{type="success"}
Defining the `foreign` key once in your `.cube` schema means `with('orders')`
just works with no options. See
[Schema relationships](/guides/schemas/relationships).
::
