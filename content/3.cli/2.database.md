## database:create

Creates the database(s) defined in your `dbcube.config.js` file.

### Usage

```bash
dbcube run database:create
```

### What It Does

This command reads your `dbcube.config.js` file and creates the database(s) defined in it.

### Prerequisites

You must have a `dbcube.config.js` file in your project root with at least one database configuration:

```typescript
// dbcube.config.js
module.exports = (config) => {
  config.addDatabase({
    name: "myapp",
    type: "mysql",       // mysql, postgres, sqlite, mongodb
    host: "localhost",
    port: 3306,
    user: "root",
    password: "password",
    database: "myapp_db"
  });
};
```

### Supported Database Types

#### MySQL

```typescript
config.addDatabase({
  name: "myapp",
  type: "mysql",
  host: "localhost",
  port: 3306,
  user: "root",
  password: "password",
  database: "myapp_db"
});
```

#### PostgreSQL

```typescript
config.addDatabase({
  name: "analytics",
  type: "postgres",
  host: "localhost",
  port: 5432,
  user: "postgres",
  password: "password",
  database: "analytics_db"
});
```

#### SQLite

```typescript
config.addDatabase({
  name: "local",
  type: "sqlite",
  path: "./database.sqlite"
});
```

#### MongoDB

```typescript
config.addDatabase({
  name: "logs",
  type: "mongodb",
  host: "mongodb://localhost:27017",
  database: "app_logs"
});
```

### Environment Variables

Use environment variables for sensitive data:

```typescript
// dbcube.config.js
require('dotenv').config();

module.exports = (config) => {
  config.addDatabase({
    name: "production",
    type: "mysql",
    host: process.env.DB_HOST,
    port: parseInt(process.env.DB_PORT),
    user: process.env.DB_USER,
    password: process.env.DB_PASSWORD,
    database: process.env.DB_NAME
  });
};
```

Then in your `.env` file:

```bash
DB_HOST=prod-db.company.com
DB_PORT=3306
DB_USER=prod_user
DB_PASSWORD=secure_password
DB_NAME=myapp_production
```

### Multiple Databases

You can configure multiple databases in the same file:

```typescript
module.exports = (config) => {
  // Primary database
  config.addDatabase({
    name: "primary",
    type: "mysql",
    host: "localhost",
    port: 3306,
    user: "root",
    password: "password",
    database: "myapp_db"
  });

  // Analytics database
  config.addDatabase({
    name: "analytics",
    type: "postgres",
    host: "analytics-server",
    port: 5432,
    user: "analytics_user",
    password: "analytics_pass",
    database: "analytics_db"
  });

  // Cache database
  config.addDatabase({
    name: "cache",
    type: "mongodb",
    host: "mongodb://localhost:27017",
    database: "cache_db"
  });
};
```

## Example Workflow

### 1. Create Configuration File

Create `dbcube.config.js` in your project root:

```typescript
module.exports = (config) => {
  config.addDatabase({
    name: "myapp",
    type: "mysql",
    host: "localhost",
    port: 3306,
    user: "root",
    password: "password",
    database: "myapp_db"
  });
};
```

### 2. Create Database

Run the command:

```bash
dbcube run database:create
```

Output:
```
✓ Database 'myapp_db' created successfully
✓ Connected to MySQL at localhost:3306
```

### 3. Verify Database

You can verify the database was created using your database client:

```bash
# For MySQL
mysql -u root -p -e "SHOW DATABASES LIKE 'myapp_db';"

# For PostgreSQL
psql -U postgres -l | grep myapp_db

# For SQLite
ls -la database.sqlite
```

## Configuration Options

### Common Options (All Database Types)

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `name` | string | Yes | Unique identifier for this database |
| `type` | string | Yes | Database type: `mysql`, `postgres`, `sqlite`, `mongodb` |
| `database` | string | Yes | Database name to create/use |

### MySQL/PostgreSQL Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `host` | string | Yes | Database server hostname |
| `port` | number | Yes | Database server port |
| `user` | string | Yes | Database username |
| `password` | string | Yes | Database password |

### SQLite Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `path` | string | Yes | Path to SQLite database file |

### MongoDB Options

| Option | Type | Required | Description |
|--------|------|----------|-------------|
| `host` | string | Yes | MongoDB connection string |
| `database` | string | Yes | Database name |

## Troubleshooting

### Database Already Exists

If the database already exists, the command will either:
- Use the existing database (most database types)
- Show a warning but continue

### Connection Errors

**Error: Can't connect to database server**

```bash
# Check if database server is running
sudo systemctl status mysql    # MySQL
sudo systemctl status postgresql  # PostgreSQL
```

**Error: Access denied**

```bash
# Verify credentials in dbcube.config.js
# Check user has CREATE DATABASE permission
mysql -u root -p -e "SHOW GRANTS;"
```

### Permission Errors

**Error: Access denied creating database**

Your database user needs CREATE DATABASE permission:

```sql
-- MySQL
GRANT CREATE ON *.* TO 'username'@'localhost';
FLUSH PRIVILEGES;

-- PostgreSQL
ALTER USER username CREATEDB;
```
