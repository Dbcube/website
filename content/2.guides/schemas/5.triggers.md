## What are Triggers?

Triggers are functions that execute automatically when specific database events occur:

- **Automatic**: Execute without manual intervention
- **Event-driven**: Respond to INSERT, UPDATE, DELETE
- **Validation**: Verify data before modifications
- **Auditing**: Record changes for traceability
- **Business Logic**: Implement complex business rules

## Supported Trigger Types

Dbcube supports 6 trigger types that cover all data lifecycle events:

### Insert Triggers

- **`@beforeAdd`**: Before inserting a record
- **`@afterAdd`**: After inserting a record

### Update Triggers

- **`@beforeUpdate`**: Before updating a record
- **`@afterUpdate`**: After updating a record

### Delete Triggers

- **`@beforeDelete`**: Before deleting a record
- **`@afterDelete`**: After deleting a record

## Basic Trigger Structure

::cube-code
---
code: |
    @database("database_name");        // Target database
    @table("table_name");              // Table that triggers the event

    @triggerType({
        name: "trigger_name";
        description: "Trigger description";
        function: ({db, oldData, newData, gdb}) => {
            // Trigger logic
        };
    });
---
::

### Available Parameters

- **`db`**: Database instance for queries
- **`oldData`**: Data before modification (UPDATE/DELETE)
- **`newData`**: New or modified data (INSERT/UPDATE)
- **`gdb`**: Global database instance

## @beforeAdd Triggers

Execute **before** inserting a new record, ideal for:

- Validating input data
- Setting default values
- Generating calculated fields
- Checking permissions

### Data Validation

::cube-code
---
code: |
    // users.trigger.cube
    @database("blog");
    @table("users");

    @beforeAdd({
        name: "validate_user";
        description: "Validate user data before creation";
        function: ({db, oldData, newData, gdb}) => {
            // Validate email format
            const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
            if (!emailRegex.test(newData.email)) {
                throw new Error('Invalid email format');
            }

            // Check if email is already in use
            const emailExists = db.table('users')
                .where('email', '=', newData.email)
                .where('status', '!=', 'deleted')
                .first();

            if (emailExists) {
                throw new Error('Email already registered');
            }

            // Set default values
            newData.created_at = new Date();
            newData.status = newData.status || 'active';
            newData.email_verified = false;

            // Generate slug automatically
            newData.slug = newData.name
                .toLowerCase()
                .replace(/[^a-z0-9]/g, '-')
                .replace(/-+/g, '-')
                .replace(/^-|-$/g, '');

            console.log(`Validating new user: ${newData.email}`);
        };
    });

### Automatic Calculations

::cube-code
---
code: |
    // products.trigger.cube
    @database("store");
    @table("products");

    @beforeAdd({
        name: "calculate_product_fields";
        description: "Calculate derived product fields";
        function: ({db, oldData, newData, gdb}) => {
            // Calculate discounted price if applicable
            if (newData.discount_percentage > 0) {
                const discount = (newData.price * newData.discount_percentage) / 100;
                newData.sale_price = newData.price - discount;
            }

            // Generate automatic SKU if not provided
            if (!newData.sku) {
                const category = db.table('categories').find(newData.category_id);
                const timestamp = Date.now();
                newData.sku = `${category.code}-${timestamp}`;
            }

            // Set estimated weight based on category
            if (!newData.weight) {
                const averageWeights = {
                    'electronics': 0.5,
                    'books': 0.3,
                    'clothing': 0.2,
                    'furniture': 15.0
                };

                const category = db.table('categories').find(newData.category_id);
                newData.weight = averageWeights[category.slug] || 1.0;
            }

            console.log(`Product created with SKU: ${newData.sku}`);
        };
    });

## @afterAdd Triggers

Execute **after** inserting a record, perfect for:

- Notifications
- Updating counters
- Creating related records
- Auditing

### Post-Creation Actions

::cube-code
---
code: |
    // posts.trigger.cube
    @database("blog");
    @table("posts");

    @afterAdd({
        name: "post_created_actions";
        description: "Actions after creating a post";
        function: ({db, oldData, newData, gdb}) => {
            // Update user's post count
            const user = db.table('users').where('id', '=', newData.user_id).first();
            if (user) {
                db.table('users')
                    .where('id', '=', newData.user_id)
                    .update({ total_posts: (user.total_posts || 0) + 1 });
            }

            // Create audit record
            db.table('audit_log').insert([{
                table: 'posts',
                record_id: newData.id,
                action: 'create',
                user_id: newData.user_id,
                timestamp: new Date(),
                data: JSON.stringify(newData)
            });

            // If post is published, notify subscribers
            if (newData.status === 'published') {
                // Get category subscribers
                const subscribers = db.table('subscriptions')
                    .where('category_id', '=', newData.category_id)
                    .where('active', '=', true)
                    .get();

                // Create notifications
                subscribers.forEach(subscriber => {
                    db.table('notifications').insert([{
                        user_id: subscriber.user_id,
                        type: 'new_post',
                        title: `New post: ${newData.title}`,
                        message: `A new post has been published in ${newData.category.name}`,
                        post_id: newData.id,
                        read: false,
                        created_at: new Date()
                    }]);
                });

                console.log(`Notifications sent for post: ${newData.title}`);
            }

            console.log(`Post created: ${newData.title} by user ${newData.user_id}`);
        };
    });

## @beforeUpdate Triggers

Execute **before** updating a record:

### Transition Validations

::cube-code
---
code: |
    // orders.trigger.cube
    @database("store");
    @table("orders");

    @beforeUpdate({
        name: "validate_status_transition";
        description: "Validate allowed status transitions";
        function: ({db, oldData, newData, gdb}) => {
            // Define valid transitions
            const validTransitions = {
                'pending': ['processing', 'cancelled'],
                'processing': ['shipped', 'cancelled'],
                'shipped': ['delivered', 'returned'],
                'delivered': ['returned'],
                'cancelled': [],
                'returned': []
            };

            // Check if status changed
            if (oldData.status !== newData.status) {
                const allowedStatuses = validTransitions[oldData.status];

                if (!allowedStatuses.includes(newData.status)) {
                    throw new Error(
                        `Invalid transition: Cannot change from '${oldData.status}' to '${newData.status}'`
                    );
                }
            }

            // Validate total changes if order is already processing
            if (oldData.status !== 'pending' && oldData.total !== newData.total) {
                throw new Error('Cannot modify total of an order that is already being processed');
            }

            // Update modification timestamp
            newData.updated_at = new Date();

            // Record who made the change if user is in session
            if (global.currentUser) {
                newData.modified_by = global.currentUser.id;
            }

            console.log(`Valid status transition: ${oldData.status} → ${newData.status}`);
        };
    });

### Dynamic Calculations

::cube-code
---
code: |
    // invoices.trigger.cube
    @database("accounting");
    @table("invoices");

    @beforeUpdate({
        name: "recalculate_totals";
        description: "Recalculate totals when items change";
        function: ({db, oldData, newData, gdb}) => {
            // Recalculate only if items or discounts changed
            const itemsChanged = JSON.stringify(oldData.items) !== JSON.stringify(newData.items);
            const discountChanged = oldData.discount !== newData.discount;

            if (itemsChanged || discountChanged) {
                let subtotal = 0;

                // Calculate subtotal from all items
                if (newData.items && Array.isArray(newData.items)) {
                    subtotal = newData.items.reduce((sum, item) => {
                        return sum + (item.quantity * item.unit_price);
                    }, 0);
                }

                // Apply discount if exists
                const discount = (subtotal * (newData.discount_percentage || 0)) / 100;
                const subtotalWithDiscount = subtotal - discount;

                // Calculate taxes (VAT)
                const taxes = subtotalWithDiscount * 0.21; // 21% VAT

                // Calculate final total
                const total = subtotalWithDiscount + taxes;

                // Update calculated fields
                newData.subtotal = subtotal.toFixed(2);
                newData.discount_applied = discount.toFixed(2);
                newData.taxes = taxes.toFixed(2);
                newData.total = total.toFixed(2);

                console.log(`Totals recalculated - Subtotal: ${newData.subtotal}, Total: ${newData.total}`);
            }
        };
    });

## @afterUpdate Triggers

Execute **after** updating a record:

### Notifications and Auditing

::cube-code
---
code: |
    // users.trigger.cube
    @database("blog");
    @table("users");

    @afterUpdate({
        name: "user_updated";
        description: "Actions after updating user";
        function: ({db, oldData, newData, gdb}) => {
            // Audit important changes
            const importantFields = ['email', 'status', 'role'];
            const changes = {};

            importantFields.forEach(field => {
                if (oldData[field] !== newData[field]) {
                    changes[field] = {
                        before: oldData[field],
                        after: newData[field]
                    };
                }
            });

            if (Object.keys(changes).length > 0) {
                db.table('user_audit').insert([{
                    user_id: newData.id,
                    changes: JSON.stringify(changes),
                    ip_address: global.clientIP || 'unknown',
                    user_agent: global.userAgent || 'unknown',
                    timestamp: new Date(),
                    modified_by: global.currentUser?.id
                }]);
            }

            // Notify status change
            if (oldData.status !== newData.status) {
                db.table('notifications').insert([{
                    user_id: newData.id,
                    type: 'status_change',
                    title: 'Your account status has changed',
                    message: `Your account status changed from '${oldData.status}' to '${newData.status}'`,
                    created_at: new Date(),
                    read: false
                }]);
            }

            // Invalidate cache if critical information changed
            if (changes.email || changes.role) {
                // Invalidate active sessions if email or role changed
                db.table('sessions')
                    .where('user_id', '=', newData.id)
                    .update({ valid: false });

                console.log(`Sessions invalidated for user ${newData.id} due to critical change`);
            }

            console.log(`User ${newData.id} updated with ${Object.keys(changes).length} changes`);
        };
    });

## @beforeDelete Triggers

Execute **before** deleting a record:

### Integrity Validations

::cube-code
---
code: |
    // categories.trigger.cube
    @database("blog");
    @table("categories");

    @beforeDelete({
        name: "validate_category_deletion";
        description: "Check if category can be deleted";
        function: ({db, oldData, newData, gdb}) => {
            // Check if there are posts using this category
            const postsCount = db.table('posts')
                .where('category_id', '=', oldData.id)
                .where('status', '!=', 'deleted')
                .count();

            if (postsCount > 0) {
                throw new Error(
                    `Cannot delete category '${oldData.name}' because it has ${postsCount} associated posts`
                );
            }

            // Check for subcategories
            const subcategoriesCount = db.table('categories')
                .where('parent_category_id', '=', oldData.id)
                .count();

            if (subcategoriesCount > 0) {
                throw new Error(
                    `Cannot delete category '${oldData.name}' because it has ${subcategoriesCount} subcategories`
                );
            }

            // Record deletion attempt for auditing
            db.table('audit_log').insert([{
                table: 'categories',
                record_id: oldData.id,
                action: 'delete_attempt',
                data: JSON.stringify(oldData),
                timestamp: new Date(),
                user_id: global.currentUser?.id
            }]);

            console.log(`Validation passed: Category '${oldData.name}' can be deleted`);
        };
    });

## @afterDelete Triggers

Execute **after** deleting a record:

### Cleanup and Notifications

::cube-code
---
code: |
    // users.trigger.cube
    @database("blog");
    @table("users");

    @afterDelete({
        name: "cleanup_deleted_user";
        description: "Cleanup after deleting user";
        function: ({db, oldData, newData, gdb}) => {
            // Clean up related data

            // 1. Anonymize user's comments
            db.table('comments')
                .where('user_id', '=', oldData.id)
                .update({
                    user_id: null,
                    anonymous_name: 'Deleted User',
                    anonymous_email: 'deleted@system.com'
                });

            // 2. Transfer posts to "system" user or delete them
            const systemUser = db.table('users')
                .where('email', '=', 'system@blog.com')
                .first();

            if (systemUser) {
                db.table('posts')
                    .where('user_id', '=', oldData.id)
                    .update({ user_id: systemUser.id });
            } else {
                // If no system user, mark posts as orphaned
                db.table('posts')
                    .where('user_id', '=', oldData.id)
                    .update({
                        user_id: null,
                        status: 'orphaned'
                    });
            }

            // 3. Delete active sessions
            db.table('sessions')
                .where('user_id', '=', oldData.id)
                .delete();

            // 4. Delete pending notifications
            db.table('notifications')
                .where('user_id', '=', oldData.id)
                .delete();

            // 5. Record complete deletion in audit log
            db.table('audit_log').insert([{
                table: 'users',
                record_id: oldData.id,
                action: 'delete_completed',
                data: JSON.stringify({
                    email: oldData.email,
                    name: oldData.name,
                    deletion_date: new Date()
                }),
                timestamp: new Date(),
                user_id: global.currentUser?.id
            }]);

            // 6. Notify administrators
            const admins = db.table('users')
                .where('role', '=', 'admin')
                .where('status', '=', 'active')
                .get();

            admins.forEach(admin => {
                db.table('notifications').insert([{
                    user_id: admin.id,
                    type: 'user_deleted',
                    title: 'User deleted from system',
                    message: `User '${oldData.email}' has been deleted from the system`,
                    created_at: new Date(),
                    read: false
                }]);
            });

            console.log(`User deleted: ${oldData.email}. Related data cleaned up.`);
        };
    });

## Advanced Practical Examples

### Inventory System with Triggers

::cube-code
---
code: |
    // products.trigger.cube
    @database("inventory");
    @table("products");

    @beforeUpdate({
        name: "stock_control";
        description: "Control stock changes and alerts";
        function: ({db, oldData, newData, gdb}) => {
            // Only process if stock changed
            if (oldData.stock !== newData.stock) {
                const difference = newData.stock - oldData.stock;

                // Record stock movement
                db.table('stock_movements').insert([{
                    product_id: newData.id,
                    type: difference > 0 ? 'in' : 'out',
                    quantity: Math.abs(difference),
                    previous_stock: oldData.stock,
                    new_stock: newData.stock,
                    timestamp: new Date(),
                    user_id: global.currentUser?.id,
                    reason: difference > 0 ? 'restock' : 'sale'
                }]);

                // Check minimum stock
                if (newData.stock <= newData.minimum_stock) {
                    // Create low stock alert
                    db.table('alerts').insert([{
                        type: 'low_stock',
                        product_id: newData.id,
                        message: `Low stock: ${newData.name} (${newData.stock} units)`,
                        level: newData.stock === 0 ? 'critical' : 'warning',
                        timestamp: new Date(),
                        resolved: false
                    }]);

                    // Notify purchasing department
                    const purchasingTeam = db.table('users')
                        .where('department', '=', 'purchasing')
                        .where('status', '=', 'active')
                        .get();

                    purchasingTeam.forEach(user => {
                        db.table('notifications').insert([{
                            user_id: user.id,
                            type: 'low_stock',
                            title: 'Low Stock Alert',
                            message: `Product '${newData.name}' has low stock (${newData.stock} units)`,
                            urgent: newData.stock === 0,
                            created_at: new Date()
                        }]);
                    });
                }

                console.log(`Stock updated: ${newData.name} (${oldData.stock} → ${newData.stock})`);
            }
        };
    });

## Cross-Database Access with gdb

The `gdb` parameter allows access to tables in other databases from within triggers:

::cube-code
---
code: |
    // sync-inventory.trigger.cube
    @database("store");
    @table("orders");

    @afterAdd({
        name: "sync_warehouse_inventory";
        description: "Update inventory in warehouse database";
        function: ({db, oldData, newData, gdb}) => {
            // Access current database (store)
            const orderItems = db.table('order_items')
                .where('order_id', '=', newData.id)
                .get();

            // Access different database (warehouse) using gdb
            orderItems.forEach(item => {
                const warehouseProduct = gdb.database("warehouse")
                    .table("products")
                    .where('sku', '=', item.sku)
                    .first();

                if (warehouseProduct) {
                    // Update stock in warehouse database
                    gdb.database("warehouse")
                        .table("products")
                        .where('sku', '=', item.sku)
                        .update({
                            available_stock: warehouseProduct.available_stock - item.quantity,
                            last_updated: new Date()
                        });
                }
            });

            console.log(`Order ${newData.id} synced with warehouse database`);
        };
    });

## Using require() for External Integrations

Triggers can use Node.js `require()` to import external modules for integrations:

::cube-code
---
code: |
    // real-time-notifications.trigger.cube
    @database("blog");
    @table("posts");

    @afterAdd({
        name: "send_socket_notification";
        description: "Send real-time notification via Socket.IO";
        function: ({db, oldData, newData, gdb}) => {
            // Import Socket.IO client
            const io = require('socket.io-client');
            const socket = io('http://localhost:3000');

            // Emit real-time event
            if (newData.status === 'published') {
                socket.emit('new-post', {
                    id: newData.id,
                    title: newData.title,
                    author_id: newData.user_id,
                    category_id: newData.category_id,
                    published_at: new Date()
                });

                console.log(`Real-time notification sent for post: ${newData.title}`);
            }

            socket.disconnect();
        };
    });

                }]);

                console.log(`Welcome email sent to ${newData.email}`);
            }).catch(error => {
                console.error(`Failed to send welcome email: ${error.message}`);
            });
        };
    });
---
::

## Best Practices

### 1. Keep Triggers Simple and Focused

::cube-code
---
code: |
    // ✅ Focused trigger for specific task
    @beforeAdd({
        name: "validate_email";
        description: "Validate email format only";
        function: ({db, oldData, newData, gdb}) => {
            const emailRegex = /^[^\s@]+@[^\s@]+\.[^\s@]+$/;
            if (!emailRegex.test(newData.email)) {
                throw new Error('Invalid email format');
            }
        };
    });

    // ❌ Trigger doing too many things
    @beforeAdd({
        name: "validate_everything";
        description: "Validate email, generate slug, update counters, send emails...";
        function: ({db, oldData, newData, gdb}) => {
            // Too many responsibilities in one trigger
        };
    });
---
::

### 2. Use Proper Error Handling

::cube-code
---
code: |
    @beforeUpdate({
        name: "validate_transition";
        description: "Validate status transition";
        function: ({db, oldData, newData, gdb}) => {
            try {
                // Validation logic
                if (invalidCondition) {
                    throw new Error('Specific error message for user');
                }
            } catch (error) {
                // Optional debugging log
                console.error(`Error in validate_transition trigger: ${error.message}`);

                // Re-throw for application to handle
                throw error;
            }
        };
    });
---
::

### 3. Document Each Trigger Well

::cube-code
---
code: |
    @beforeAdd({
        name: "generate_product_slug";
        description: "Generate unique slug for product based on name";
        // PURPOSE: Automate creation of SEO-friendly URLs
        // WHEN: Before inserting a product if it doesn't have a slug
        // DEPENDENCIES: Products table must have 'slug' column
        // EFFECTS: Modifies newData.slug
        function: ({db, oldData, newData, gdb}) => {
            if (!newData.slug) {
                let baseSlug = newData.name
                    .toLowerCase()
                    .replace(/[^a-z0-9]/g, '-')
                    .replace(/-+/g, '-')
                    .replace(/^-|-$/g, '');

                // Check uniqueness
                let counter = 0;
                let finalSlug = baseSlug;

                while (db.table('products').where('slug', '=', finalSlug).first() !== null) {
                    counter++;
                    finalSlug = `${baseSlug}-${counter}`;
                }

                newData.slug = finalSlug;
            }
        };
    });
---
::

### 4. Avoid Recursive Triggers

::cube-code
---
code: |
    // ❌ Trigger that can cause recursion
    @afterUpdate({
        name: "update_timestamp",
        function: ({db, oldData, newData, gdb}) => {
            // This causes infinite recursion!
            db.table('posts')
                .where('id', '=', newData.id)
                .update({ last_modified: new Date() });
        };
    });

    // ✅ Use @beforeUpdate to modify current record
    @beforeUpdate({
        name: "update_timestamp",
        function: ({db, oldData, newData, gdb}) => {
            // Modify newData directly, no UPDATE query
            newData.last_modified = new Date();
        };
    });
---
::

## Troubleshooting

### 1. Error in Trigger Blocks Operations

::cube-code
---
code: |
    // Problem: Trigger error prevents all operations
    // Solution: Use try-catch for non-critical errors

    @beforeAdd({
        name: "send_notification",
        function: ({db, oldData, newData, gdb}) => {
            try {
                // Logic that might fail (e.g., external service)
                sendWelcomeEmail(newData.email);
            } catch (error) {
                // Log error but don't stop operation
                console.error('Error sending email:', error.message);
                // Don't throw for non-critical errors
            }

            // Critical validations should still throw
            if (!newData.email) {
                throw new Error('Email is required');
            }
        };
    });
---
::

### 2. Slow Trigger Affects Performance

::cube-code
---
code: |
    // Problem: Complex queries in triggers
    // Solution: Optimize queries or use async triggers

    @afterAdd({
        name: "update_statistics",
        function: ({db, oldData, newData, gdb}) => {
            // ❌ Slow query in synchronous trigger
            // const stats = db.table('posts')
            //     .join('users', 'users.id', '=', 'posts.user_id')
            //     .groupBy('category_id')
            //     .get();

            // ✅ Simple and fast update
            const stat = db.table('statistics')
                .where('type', '=', 'posts_by_category')
                .where('category_id', '=', newData.category_id)
                .first();

            if (stat) {
                db.table('statistics')
                    .where('type', '=', 'posts_by_category')
                    .where('category_id', '=', newData.category_id)
                    .update({ counter: (stat.counter || 0) + 1 });
            }
        };
    });
