The official **Dbcube skill** teaches [Claude Code](https://claude.com/claude-code)
the real Dbcube API, so it writes correct code instead of inventing methods. It
covers `.cube` files (`.table.cube`, `.seeder.cube`, `.trigger.cube`,
`.alter.cube`), the `dbcube.config.js`, the query builder (transactions,
relations, `raw`), the CLI, and cloud config (Supabase, Turso, PlanetScale,
Atlas).

It lives in its own public repo:
[**github.com/Dbcube/skills**](https://github.com/Dbcube/skills).

## Install (recommended)

Inside Claude Code, add the marketplace and install the plugin:

```text
/plugin marketplace add Dbcube/skills
/plugin install dbcube@dbcube
```

Restart Claude Code if prompted. The skill activates **automatically** whenever
you work in a Dbcube project — detected by a `dbcube.config.js`, a `dbcube/`
folder with `.cube` files, or `dbcube` / `@dbcube/*` in `package.json`.

## Install (manual)

Prefer to vendor it? Clone the repo and copy the skill into your personal or
project skills folder:

```bash
git clone https://github.com/Dbcube/skills
# personal (all your projects)
cp -r skills/plugins/dbcube/skills/dbcube ~/.claude/skills/dbcube
# or project-local (committed, shared with your team)
cp -r skills/plugins/dbcube/skills/dbcube .claude/skills/dbcube
```

## Verify it works

Ask Claude Code something like *"how do I write an update query in Dbcube?"* — it
should answer with `db.table(...).where(col, op, val).update({...})` and remind
you that `update()` requires a `where()`.

::callout{type="info"}
The skill is a curated snapshot plus pointers to the **installed package** —
when something isn't covered it reads the type declarations in
`node_modules/@dbcube/*` and your generated `dbcube/types/index.ts`, so it stays
accurate to your exact version.
::
