Everything executed through the `trx` connection commits **atomically**. Any
thrown error rolls the whole transaction back automatically. Works on MySQL,
PostgreSQL, SQLite and MongoDB (replica set). (`examples/05-transactions.js`)

## Interactive transaction

```ts
// A money transfer — both updates commit together or not at all
await db.transaction(async (trx) => {
  await trx.table('users').where('id', '=', 1).decrement('balance', 200);
  await trx.table('users').where('id', '=', 2).increment('balance', 200);
});
```

## Automatic rollback on throw

Reads inside the transaction see its own uncommitted changes, so you can guard
invariants and abort:

```ts
try {
  await db.transaction(async (trx) => {
    await trx.table('users').where('id', '=', 1).decrement('balance', 999999);

    const payer = await trx.table('users').find(1); // sees the uncommitted decrement
    if (payer.balance < 0) {
      throw new Error('Insufficient funds');         // → automatic ROLLBACK
    }

    await trx.table('users').where('id', '=', 2).increment('balance', 999999);
  });
} catch (e) {
  console.log('transaction aborted ✅ →', e.message);
}
// Balance is untouched — the whole transaction rolled back.
```

## `batch()` — the single-round-trip form

When a transaction is a pure sequence of writes (no intermediate reads), use
`db.batch()`. Dbcube ships the entire `BEGIN … COMMIT` as **one network
round-trip** instead of one per statement — this is why Dbcube's transactions
are the fastest in the [benchmarks](/performance/benchmarks).

```ts
await db.batch((b) => {
  b.table('users').where('id', '=', 1).decrement('balance', 200);
  b.table('users').where('id', '=', 2).increment('balance', 200);
});
```

| Use | When |
|---|---|
| `db.transaction(cb)` | You need to **read** inside the transaction and branch on results |
| `db.batch(cb)` | A pure **sequence of writes** — fastest, one round-trip |

::callout{type="warning"}
MongoDB transactions require a **replica set** (a single-node one is enough:
`mongod --replSet rs0` + `rs.initiate()`). Without it, `transaction()` throws a
clear error.
::
