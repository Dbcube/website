Aggregations **execute immediately and return a value** — they are not chainable
with `.first()` / `.get()`. (`examples/04-aggregations.js`)

## The five aggregates

```ts
await db.table('users').count();                                  // → number
await db.table('users').where('status', '=', 'active').count();   // count with filter
await db.table('users').sum('balance');
await db.table('users').avg('age');
await db.table('users').max('age');
await db.table('users').min('age');
```

## `exists()` — cheaper than `count() > 0`

```ts
// SELECT 1 ... LIMIT 1 → boolean
const hasElders = await db.table('users').where('age', '>', 80).exists();
```

## Grouping

```ts
const perStatus = await db.table('users')
  .selectRaw(['status', 'COUNT(*) AS n'])
  .groupBy('status')
  .having('n', '>', 0)
  .get();
// → [{ status: 'active', n: 4 }, { status: 'inactive', n: 1 }]
```

## Pagination with metadata

`paginate(page, perPage)` returns items plus the metadata you need to render a
pager:

```ts
const page = await db.table('users').orderBy('id', 'ASC').paginate(1, 2);
// {
//   items: [...],
//   total, totalPages, currentPage,
//   hasNext, hasPrev
// }
console.log(`page 1/${page.totalPages}`, page.items, 'hasNext:', page.hasNext);
```

## Streaming large tables with `chunk()`

Process millions of rows without loading them all into memory. Return `false`
from the callback to stop early.

```ts
let processed = 0;
await db.table('users').chunk(2, (rows, pageNum) => {
  processed += rows.length;
  console.log(`chunk ${pageNum}: ${rows.length} rows`);
  // return false; // → stop iterating
});
```

::callout{type="info"}
`paginate()` is for UI pagers (it computes totals); `chunk()` is for batch jobs
(it never computes a total, just walks the table page by page).
::
