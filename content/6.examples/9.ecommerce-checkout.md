Checkout is the textbook "must be correct" path: money moves, stock changes, and
a half-finished order is a bug. This is how you do it with Dbcube — typed, and
atomic by construction.

## The types

```ts
// dbcube/types.ts  (generated)
export interface Product {
  id: number;
  name: string;
  price_cents: number;
  stock: number;
}

export interface Customer {
  id: number;
  email: string;
  wallet_cents: number;
}

export interface Order {
  id: number;
  customer_id: number;
  total_cents: number;
  status: "paid" | "cancelled";
  created_at: string;
}

export interface OrderItem {
  id: number;
  order_id: number;
  product_id: number;
  qty: number;
  unit_price_cents: number;
}
```

## The checkout — one atomic transaction

Everything inside `transaction()` commits together. Any thrown error (out of
stock, insufficient funds) rolls **everything** back — no orphan order, no lost
stock, no double charge.

```ts
import { dbcube } from "dbcube";
import type { Product, Customer, Order } from "./dbcube/types";

const db = dbcube.database("shop");

interface CartLine { productId: number; qty: number; }

export async function checkout(customerId: number, cart: CartLine[]): Promise<Order> {
  if (cart.length === 0) throw new Error("Cart is empty");

  return db.transaction(async (trx) => {
    // 1. Load the products being bought (reads see uncommitted changes in-tx)
    const ids = cart.map((l) => l.productId);
    const products = await trx.table<Product>("products").whereIn("id", ids).get();
    const byId = new Map(products.map((p) => [p.id, p]));

    // 2. Validate stock and compute the total
    let total = 0;
    for (const line of cart) {
      const product = byId.get(line.productId);
      if (!product) throw new Error(`Product ${line.productId} not found`);
      if (product.stock < line.qty) {
        throw new Error(`Out of stock: ${product.name}`); // → ROLLBACK
      }
      total += product.price_cents * line.qty;
    }

    // 3. Check funds and charge the wallet atomically
    const customer = await trx.table<Customer>("customers").find(customerId);
    if (!customer) throw new Error("Customer not found");
    if (customer.wallet_cents < total) {
      throw new Error("Insufficient funds"); // → ROLLBACK, nothing charged
    }
    await trx.table<Customer>("customers")
      .where("id", "=", customerId)
      .decrement("wallet_cents", total);

    // 4. Create the order
    const [order] = await trx.table<Order>("orders").insert([
      { customer_id: customerId, total_cents: total, status: "paid" },
    ]);

    // 5. Create line items + decrement inventory
    for (const line of cart) {
      const product = byId.get(line.productId)!;
      await trx.table<OrderItem>("order_items").insert([
        {
          order_id: order.id,
          product_id: product.id,
          qty: line.qty,
          unit_price_cents: product.price_cents,
        },
      ]);
      await trx.table<Product>("products")
        .where("id", "=", product.id)
        .decrement("stock", line.qty);
    }

    return order; // committed only if every step above succeeded
  });
}
```

## Reading an order back with its items

```ts
export async function getOrder(orderId: number) {
  return db.table<Order>("orders")
    .where("id", "=", orderId)
    .with("items", { table: "order_items", foreignKey: "order_id", type: "many" })
    .with("customer", { table: "customers", foreignKey: "customer_id", type: "one" })
    .first();
}
```

## The fast path: `db.batch()` for known-good writes

When you've already validated and just need to commit several writes as one unit,
`db.batch()` ships begin + all writes + commit in **a single network round-trip**
— the cheapest way to run an atomic sequence.

```ts
// e.g. refund: credit the wallet and flip the order status, atomically, in 1 RTT
export async function refund(orderId: number, customerId: number, amountCents: number) {
  await db.batch((b) => {
    b.table("customers").where("id", "=", customerId).increment("wallet_cents", amountCents);
    b.table("orders").where("id", "=", orderId).update({ status: "cancelled" });
  });
}
```

::callout{type="warning"}
Use `transaction()` (interactive) when you need to **read intermediate results**
and branch on them — like the stock/funds checks above. Use `batch()` when the
writes are already decided: it's faster, but the callback can't read.
::

::callout{type="success"}
Because stock and wallet moves happen inside the transaction with row-level
locking, two concurrent checkouts for the last item in stock can't both succeed —
one commits, the other rolls back with "Out of stock". Correctness without manual
locks.
::
